# 翻訳・ローカライズマニュアル（Localization）

## このスキルの目的
Seep Logos の全コンテンツのローカライズ・翻訳作業の標準プロセス・品質基準・用語管理を定めたマニュアル。
IP世界観を損なわず、各地域の言語・文化に自然に届けることを目指す。

---

## 1. ローカライズの基本原則

### 世界観を守る翻訳
固有名詞・世界観用語は「訳す」のではなく「届ける」という意識を持つ。
- 「魔法」を単純に "magic" と訳すのではなく、その世界観における固有の概念として翻訳する
- キャラクターの口調・個性が翻訳後も失われないよう配慮する

### 直訳より伝達
「原文に忠実な直訳」より「ターゲット言語で自然な表現」を優先する。
ただし、世界観・IP固有の表現はスタイルガイドに従い保持する。

### 品質の3段階
```
Level 1（スピード重視）: 機械翻訳 + 最小限のポストエディット
Level 2（品質標準）: プロ翻訳者による翻訳 + 用語チェック
Level 3（最高品質）: プロ翻訳者 + ネイティブレビュー + 文化適応チェック
```
コンテンツの重要度によりレベルを使い分ける。

---

## 2. ローカライズ対象コンテンツの優先順位

| 優先度 | コンテンツ種別 | 推奨レベル |
|---|---|---|
| 最高 | ゲームシナリオ（メイン） | Level 3 |
| 高 | プレスリリース・公式発表文 | Level 3 |
| 高 | Webサイト（トップ・作品説明） | Level 2〜3 |
| 中 | ゲームシステムテキスト・UI | Level 2 |
| 中 | SNS投稿 | Level 2 |
| 低 | 内部資料・社内文書 | Level 1 |

---

## 3. 用語集・スタイルガイドの管理

### 用語集の構成
```
| 日本語用語 | 英語訳 | 中国語（繁体）訳 | 韓国語訳 | 注記・文脈説明 |
```

### スタイルガイドに定める項目
- 敬語・丁寧語の扱い（ゲームキャラは何語で話すか）
- 句読点・記号の使い方（言語別ルール）
- 数字の表記（全角/半角/アラビア数字/漢数字）
- キャラクター名の表記（カタカナ→アルファベット変換ルール）
- ブランド名・固有名詞の扱い（翻訳する/しない）

### 用語集の管理ルール
- IP単位で用語集を管理する
- 用語の追加・変更は localization-director が承認する
- 翻訳者全員が同一バージョンの用語集を使用する

---

## 4. 翻訳プロセスの標準フロー

### Step 1: 前処理
- 翻訳対象テキストの抽出（ゲームの場合はCSV/Excel形式に変換）
- 文字数カウント（工数・費用の見積もりに使用）
- 翻訳禁止用語・固有名詞の確認

### Step 2: 翻訳
- 用語集・スタイルガイドを参照しながら翻訳
- 機械翻訳使用時は必ずポストエディットを実施
- 文字数制限がある場合（UI等）は制限内に収める

### Step 3: 品質チェック
- 用語の統一性確認（用語集との照合）
- スペルチェック・文法チェック
- 文字数制限の確認

### Step 4: レビュー（Level 3 の場合）
- ネイティブチェック
- 文化適応確認（cultural-adaptation と連携）
- 最終承認

### Step 5: 納品・フィードバック
- 翻訳テキストをゲームエンジン・CMS等に実装
- 実装後の表示確認（文字切れ・レイアウト崩れ等）

---

## 5. 言語別注意事項

### 英語
- アメリカ英語・イギリス英語の統一（原則アメリカ英語）
- 略語・スラングはターゲット年齢層に合わせて調整

### 繁体字中国語（台湾・香港）
- 簡体字と繁体字は別ファイルで管理する
- 台湾と香港では一部用語が異なる場合がある

### 韓国語
- 敬語体（합니다체）と打ち解け体（해요体）を用途別に使い分け
- ゲームシナリオはキャラクター設定に合わせた口調を維持

---

## 6. 品質チェックリスト

- [ ] 用語集の表記に統一されているか
- [ ] キャラクターの口調・個性が翻訳後も維持されているか
- [ ] 文字数制限（UI枠等）をオーバーしていないか
- [ ] 文化的に問題のある表現がないか
- [ ] 翻訳前後で意味が変わっていないか
- [ ] 機密情報の取り扱い（翻訳ツールへの入力可否）が適切か

## 7. よくあるミス & NG例

### NG例1：世界観用語の無秩序な翻訳
```
NG: 同じ魔法の名前が「〇〇の術」「〇〇スキル」「〇〇魔法」とシーンごとに変わる
OK: 用語集を作成し、全翻訳者が同じ訳語を使う
```

### NG例2：文字数超過
```
NG: 「決定」（2文字）の英訳が "Confirm" で枠に収まらない
OK: 事前に各UI枠の文字数制限を取得し、翻訳時に考慮する
```

### NG例3：機械翻訳そのまま納品
```
NG: キャラクターの一人称「俺」「わたくし」が全て "I" に統一される
OK: キャラクター設定に合わせた個性ある翻訳にポストエディットする
```
