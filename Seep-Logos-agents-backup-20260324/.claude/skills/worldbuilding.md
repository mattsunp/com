# 世界観設定構築マニュアル（Worldbuilding）

## このスキルの目的
Seep Logos の全IPにおける世界観設定の構築プロセス・管理方法・品質基準を定めたマニュアル。
「無限の"世界"を創生する」というミッションの具体的な実践方法を示す。

---

## 1. 世界観構築の哲学

### 「生きている世界」を作る
設定のための設定ではなく、「その世界で生きている人の日常」まで考える。
主人公が登場しない場所でも世界が動いている——そのリアリティが没入感を生む。

### 「なぜ」から始める
設定を作る時は「どんな」より「なぜ」を先に問う。
- なぜこの国が繁栄しているのか
- なぜこの種族は他と対立するのか
- なぜ主人公はこの世界に生まれたのか

### 拡張可能な設計
シリーズ展開・メディアミックスを見据え、設定に「拡張の余地」を残す。
全てを決めきらず、後のクリエイターが想像力を発揮できる余白を持つ。

## 2. 世界観設定の7レイヤー

### Layer 1: 世界の基本原則
- 物理法則（重力・時間・因果の特殊設定等）
- 魔法・技術・エネルギー体系（存在するか・どう機能するか）
- 神・超自然的存在の有無と役割

### Layer 2: 地理・地形
- 大陸・海・山・都市の配置
- 気候・環境（気象・生態系）
- 主要な地名と各地の特徴・雰囲気

### Layer 3: 歴史・年表
- 世界の創造神話・起源
- 重要な歴史的事件（戦争・革命・自然災害等）
- 現在に至るまでの政治的変遷

### Layer 4: 勢力・組織
- 国家・王国・共和国の一覧
- ギルド・宗教組織・犯罪組織等
- 各勢力の目的・関係（同盟・対立・中立）

### Layer 5: 種族・民族
- 登場する種族の一覧（身体的特徴・寿命・能力）
- 各種族の文化・価値観・タブー
- 種族間の歴史的関係性

### Layer 6: 文化・日常
- 食事・言語・音楽・芸術
- 宗教・信仰・神話
- 衣服・建築様式・移動手段

### Layer 7: 経済・技術
- 通貨制度・交易
- 技術レベル（時代感）
- 産業・資源

---

## 3. 設定資料の管理ルール

### バージョン管理
- 設定資料には必ずバージョン番号と更新日を記載
- 変更点は「更新履歴」セクションに記録
- 公式設定は `[確定]`、検討中は `[検討中]`、廃案は `[廃案]` でラベル管理

### 優先度管理
```
【優先度の定義】
Tier 1（コア設定）: 変更すると世界観全体に影響する根本設定
Tier 2（主要設定）: 物語の進行に関わる重要設定
Tier 3（拡張設定）: 世界の深みを出す補足設定
Tier 4（余白設定）: 後のクリエイターが拡張可能なエリア
```

### 用語集の整備
- 世界固有の名詞は全て用語集に登録する
- 英語・ローカライズ用の翻訳注記も追記する
- 読み方・略称も記載する

## 4. 設定構築チェックリスト

### 整合性チェック
- [ ] 世界の基本原則（物理法則・魔法体系等）に矛盾がないか
- [ ] 歴史年表と現在の設定が整合しているか
- [ ] 各勢力の目的・行動が論理的に一貫しているか
- [ ] 種族・文化の設定がステレオタイプにならないよう工夫されているか

### 拡張性チェック
- [ ] シリーズ続編・スピンオフの余地があるか
- [ ] 異なる媒体（ゲーム/アニメ/コミック）に展開可能か
- [ ] クリエイターが「ここで何か作りたい」と感じられるか

### 文化的配慮チェック
- [ ] 実在の宗教・文化・民族を侮辱するような設定がないか
- [ ] ローカライズ時に問題になりそうな設定がないか

## 6. よくあるミス & NG例

### NG例1：整合性のない設定
```
NG: 「電気のない中世設定なのに、突然キャラがスマートフォンを使う」
OK: 設定内の技術レベルを定義し、全クリエイターが参照できるよう明文化する
```

### NG例2：説明のつかない「なんとなくすごい存在」
```
NG: 「伝説の最強勇者。理由は特になし。とにかく強い」
OK: 「〇〇の秘儀を習得した唯一の生存者。習得には〜という代償がある」
```

### NG例3：設定の肥大化
```
NG: 物語と無関係な設定を無制限に増やし、資料が読まれなくなる
OK: Tier 管理で「物語に関係する設定」と「背景設定」を分けて管理する
```
