## 全作品共通の最上位コンセプト

> **没入感** — あらゆるアウトプットは「受け手がその世界に没入できるか」を第一の判断軸とする。没入を妨げる表現・設計・発言は、たとえ便利であっても採用しない。

**没入感は「面白さ」と同義ではない。** 没入感とは受け手が世界に引き込まれ、自分と作品の境界が薄れる状態。面白さはその主要な経路のひとつだが、唯一ではない。受け手は「驚き」「確認」「委託」など異なる経路から面白さにアクセスする。

**面白さとビジネス成立性は順列のない並走義務である。** どの入り口から始まっても、もう一方の問いを即座に立て、どちらも最後まで問い続ける。機能・工数・予算だけを成立させたエンターテインメントの企画は必ず失敗する。「面白いかどうか分からない」は思考の終点ではなく、仮説を立て続けるための出発点である。

## セッション管理ルール

**トピック混在の検知（必須）：**
ユーザーの新しい依頼が現在進行中のトピックと明らかに異なると判断した場合、着手前に以下を宣言する：

> 「新しいトピックです。[現在のトピック名] の決定事項を書記に記録してから進めますか？」

ユーザーが了承した場合、recorder を呼んで現トピックの決定事項・未解決事項をログに残してから次のトピックへ移る。

**判断基準：**
- 現在のトピック（例：天才化プロジェクト）と無関係な新しい依頼が来た場合 → 宣言する
- 現在のトピックの延長・補足・深掘りの場合 → そのまま続ける
- 会話の冒頭（トピックがまだない状態）では宣言不要

## ディレクトリ構成

```
Seep-Logos-agents/
├── minutes/        # 議事録・稼働ログ（recorder / 全部門横断）
├── seeds/          # タイトル未確定の探索的制作物
│   ├── plots/
│   ├── scenarios/
│   └── worldbuilding/
├── titles/         # タイトル確定済みの作品単位（全部門横断）
├── proposals/      # 企画書・新規IP立案（保存先検討中）
├── docs/           # 詳細ルール・仕様ドキュメント（このファイルから参照）
├── creators/
│   └── _private/   # 【機密】個人情報・契約情報
└── marketing/
    ├── _private/   # 【機密】営業先・応募者情報
    └── ...
```

**機密ルール：** `_private/` は `.gitignore` により非公開。個人情報・契約情報は必ず `_private/` に保存。

**seeds → titles 昇格：** プロデューサーの判断により `titles/[作品名]/` 以下に移動。
