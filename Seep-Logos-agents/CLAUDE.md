## 全作品共通の最上位コンセプト

> **没入感** — あらゆるアウトプットは「受け手がその世界に没入できるか」を第一の判断軸とする。没入を妨げる表現・設計・発言は、たとえ便利であっても採用しない。

**没入感は「面白さ」と同義ではない。** 没入感とは受け手が世界に引き込まれ、自分と作品の境界が薄れる状態。面白さはその主要な経路のひとつだが、唯一ではない。受け手は「驚き」「確認」「委託」など異なる経路から面白さにアクセスする。

**面白さとビジネス成立性は順列のない並走義務である。** どの入り口から始まっても、もう一方の問いを即座に立て、どちらも最後まで問い続ける。機能・工数・予算だけを成立させたエンターテインメントの企画は必ず失敗する。「面白いかどうか分からない」は思考の終点ではなく、仮説を立て続けるための出発点である。

## 自動ルーティングルール

```
ユーザーからの指示 → キーワードマッチング → リード部門を判断 → エージェント起動
```

**複合タスク例：**
- 新IP発表のプレスリリース → planning-developer × press-release-writer × pr-communications
- ゲームUI設計 → art-director × game-ui-designer × ui-engineering
- 海外ローカライズ → localization-director × localization-specialist × cultural-adaptation
- 世界観構築 → worldbuilding-creator × scenario-writer × art-director
- 外部クライアント提案 → sales-director × project-manager

**優先度ルール：**
1. 外部公開コンテンツは必ずブランドルールチェック（→ `docs/brand-rules.md`）
2. 未発表情報を含む場合は機密フラグを立てて出力に注記
3. 複数エージェントが連携する場合、最初に起動したエージェントがリードを取る

## ディレクトリ構成

```
Seep-Logos-agents/
├── minutes/        # 議事録・稼働ログ（secretary / 全部門横断）
├── seeds/          # タイトル未確定の探索的制作物
│   ├── plots/
│   ├── scenarios/
│   └── worldbuilding/
├── titles/         # タイトル確定済みの作品単位（全部門横断）
├── proposals/      # 企画書・新規IP立案（保存先検討中）
├── docs/           # 詳細ルール・仕様ドキュメント（このファイルから参照）
├── creators/
│   └── _private/   # 【機密】個人情報・契約情報
└── marketing/
    ├── _private/   # 【機密】営業先・応募者情報
    └── ...
```

**機密ルール：** `_private/` は `.gitignore` により非公開。個人情報・契約情報は必ず `_private/` に保存。

**seeds → titles 昇格：** プロデューサーの判断により `titles/[作品名]/` 以下に移動。
