# Seep Logos — Web ビジュアルスタイルガイド

> **逆引き定義:** `web/templates/pattern-A` 〜 `pattern-P`（16パターン）の実装から抽出。
> 2026-04-04 策定

---

## 基本思想

Seep Logos の Web デザインは**「浸透（Seep）」**を体現する。

- 世界観と現実の境界は「壁」ではなく「息継ぎ」
- デザインは主張しない。受け手が気づかないうちに引き込まれる
- 暗闇の中に世界が浮かび上がる感覚を基調にする

---

## 1. カラーパレット

### 1-1. 背景色（全パターン共通の基底）

すべてのパターンが暗色背景を採用している。基本的に以下の範囲内から選択する。

| 役割 | 値の範囲 | 代表値 |
|---|---|---|
| 最暗部（ヒーロー・主背景） | #000 〜 #0a0a0a | `#080810` |
| セクション背景 | #0d0d0d 〜 #1a1a1a | `#0f0f17` |
| カード/パネル | #1a1a1a 〜 #242424 | `#1a1a2a` |

**禁止:** 白背景・薄いグレー背景の使用。明度が高い背景は没入感を破壊する。

### 1-2. テキスト色

| 役割 | 値 | 用途 |
|---|---|---|
| プライマリ | `#e8e3da` | 本文・見出し（温かみのある白） |
| セカンダリ | `#888899` | キャプション・日付・補足 |
| ディム | `#555566` | ラベル・タグ・非アクティブ |

**禁止:** 純白（`#ffffff`）の多用。冷たく浮いて見え、没入感が失われる。

### 1-3. アクセントカラー（パターンタイプ別）

アクセントカラーはパターンの「性格」によって選択する。IP ごとに決定する。

| 系統 | 代表値 | 向いているパターン | 世界観の性格 |
|---|---|---|---|
| ゴールド | `#c8b87a` / `#d4a843` | N, P, M, I | 叙事的・歴史的・重厚 |
| シアン | `#00e5cc` | H | SF・機械的・クール |
| パープル + ホットピンク | `#6c3fd5` + `#ff6b9d` | K | ポップ・キャラクター主体 |
| マルチカラー | 各カテゴリ色 | D, O | 情報整理・ポータル型 |
| アクセントなし（モノクロ） | — | A, B, C | ミニマル・ティザー・エディトリアル |

**Seep Logos の多作品展開において:** アクセントは IP 固有の色とし、スタジオ共通色は定義しない。スタジオのアイデンティティは「暗さ・余白・浸透感」という空間的特性で表現する。

## 2. タイポグラフィ

### 2-1. フォント使い分けの原則

| 役割 | フォント | 用途 |
|---|---|---|
| 物語性のある見出し | Georgia, Yu Mincho, Hiragino Mincho | タイトル・章タイトル・世界観テキスト |
| UI・情報整理系 | -apple-system, Helvetica Neue | ナビ・ボタン・カード見出し |
| メタデータ・ラベル | monospace (Courier New) | 日付・タグ・ナンバリング・コード |

**選択基準:**
- 世界観の語りが主役 → Serif（Georgia 系）
- 情報整理・操作が主役 → Sans-serif（Helvetica 系）
- 両者を混在させる場合: 見出し Serif + UI Sans-serif が基本構成

### 2-2. サイズと字間

| 要素 | サイズ | 字間 |
|---|---|---|
| ヒーロータイトル | `clamp(2rem, 8vw, 6rem)` | 通常〜やや詰め |
| セクション見出し | `clamp(1.4rem, 3vw, 2.8rem)` | 通常 |
| 本文 | `15–16px` | 通常 |
| ラベル・タグ | `9–11px` | `0.2–0.6em`（大文字推奨） |

**禁止:** ラベルに比例フォントを使う、タイトルに monospace を使う（意図的な演出の場合を除く）。

## 3. レイアウト

### 3-1. 共通構造

- **ヒーローセクション:** 最低 `min-height: 100vh`。フレックスボックスで中央配置が基本
- **コンテンツ幅:** `max-width: 800–1200px`（パターンにより異なる）。テキスト主体のパターンは `660px` まで絞る
- **サイドパディング:** `clamp(1.5rem, 5vw, 4rem)` を基本値とする

### 3-2. グリッド選択

| グリッド | 用途 |
|---|---|
| 2カラム交互 | 世界観説明・章の交互展開（B, C, E, P） |
| 3カラム | キャラクター・特徴紹介（B, G, H, I） |
| 4カラム | キャラクター図鑑（G, K） |
| ベントー（12カラム） | 情報ダッシュボード（D） |
| 非対称タイル（2fr + 1fr）| ハブ型・ポータル（J） |
| シングルカラム中央寄せ | 読み物・章構成（N, P） |

### 3-3. 角丸の使い方

- **基本は角丸なし（0px）:** A〜H, I, J, M, N, P はシャープなエッジ
- **角丸あり（8〜24px）:** ポップ・ソーシャルゲーム系（K, L, O）のみ
- **判断基準:** 「かわいさ・親しみやすさ」が必要なIPのみ角丸を採用する

---

## 4. ボーダー・区切り

- 基本: `1px solid rgba(255,255,255,0.08)` 前後の細い線
- プレースホルダー: `2px dashed rgba(255,255,255,0.15)`（画像未挿入領域）
- 区切り線（SNS ルールと同様）: 明示的な区切り線より**余白による分離**を優先する

---

## 5. アニメーション・トランジション

### 5-1. 基本原則

- **遅くて静かな動き:** 没入感を壊すのは速すぎるアニメーション
- `ease` または `cubic-bezier(0.4, 0, 0.2, 1)` を基本イージングとする
- バウンス系（`spring`、`bounce`）は使用しない（K のみ例外）

### 5-2. フェードイン（スクロールトリガー）

```css
/* 共通フェードイン */
opacity: 0 → 1
transform: translateY(20px) → translateY(0)
duration: 0.7–0.9s
delay: 0.1–0.8s（要素ごとにずらす）
```

- `IntersectionObserver` で実装
- 一度表示されたら再アニメーションしない

### 5-3. ホバー効果

- ボーダー色の遷移（`0.2–0.3s ease`）
- `box-shadow` のグロー（`0 0 20px rgba(accent, 0.3)`）
- スケールは使わない（使う場合は `scale(1.02)` 程度）

### 5-4. ナビゲーション

- ドットナビ（チャプター進行）: パルスアニメーション `0.6s ease-in-out`
- 固定サイドナビ: `72px` 幅、縦にエリアマーカーを並べる（M, P 型）

## 6. プレースホルダー（開発時）

実画像未挿入の領域は以下で統一する。

```css
background: #1a1a2a;
border: 2px dashed rgba(255,255,255,0.12);
font-family: monospace;
font-size: 11px;
letter-spacing: 0.3em;
color: #444;
/* 表示テキスト例: IMAGE / VIDEO / CHARACTER ART */
```

## 8. frontend-design スキルとの併用ルール

`frontend-design` スキルは大胆な美的選択を促す。このガイドと組み合わせる場合:

1. **背景色は暗色に固定** — スキルが明るいパレットを提案しても採用しない
2. **既存パターンに近い型を選ぶ場合は型名を明示** — 「pattern-N に近い構成で」と指定することでスキルの方向性を絞れる
3. **アクセントカラーは IP 側で決定してから指示する** — スキルに任せると毎回バラつく
4. **「Seep Logos らしさ」の最終判断基準:** 受け手がその世界に浸透できるか（没入感）
