## フェーズ別 投稿案（時系列）

### フェーズ1：公式アカウント開設〜最初のティザー

#### 設計方針
- 世界観の断片のみを届け、ユーザーの「これは何？」という好奇心を最大化する
- タイトル名・ジャンル・プラットフォームはいっさい公開しない
- アカウント自体の存在感を作ることが最優先

---

#### 投稿1：公式アカウント開設・第1投稿（Week 1 / 月曜 20:00）

【プラットフォーム：X（Twitter）】

```
Umbrella 公式アカウントです。

いつか、この場所から何かをお届けできる日を楽しみにしています。

#Umbrella
```

> 設計意図：アカウントの存在を示す最小限の宣言。「Umbrella」という名前だけを残し、ジャンルも内容も伏せる。フォロワーに「名前だけ知っている状態」を作ることで、次の情報への期待値を育てる。

投稿タイミング推奨：月曜 20:00〜21:00
参照テンプレート：Phase 1 / バリエーション A（シンプル予告型）を簡略化して使用

---

#### 投稿2：最初のティザー投稿 — 「何かが始まる」（Week 2 / 木曜 21:00）

【プラットフォーム：X（Twitter）】

```
世界には、まだ名前のついていないものがある。

#Umbrella
```

> 設計意図：世界観の「断片」を詩的な一文で届ける。情報ゼロのまま「空気感」だけを先行させる。リツイートより「スクショ保存・引用」を狙う文体。CTAや告知文は一切添えず、詩を単独で立たせることで没入感を守る。

投稿タイミング推奨：木曜 21:00
参照テンプレート：Phase 1 / バリエーション A（シンプル予告型）、世界観系定型フレーズを組み合わせ

【プラットフォーム：Instagram（同日 / ビジュアル先行）】

キャプション：
```
世界には、まだ名前のついていないものがある。

#Umbrella
```

ビジュアル指示：
- 暗く静かな背景に、光の断片（雨・光・影のいずれか）を使ったティザービジュアル
- テキストは最小限。「Umbrella」のロゴタイプのみを配置する構成を推奨
- カラーパレットはグレー・ネイビー系で統一。ブランドの「世界観の静けさ」を表現する

### フェーズ2：ティザー継続〜タイトル解禁

#### 設計方針
- ティザー期の「断片投稿」を週1〜2回継続しながら、徐々に「何かが近づいている」感を演出する
- タイトル解禁のタイミングは「ティザービジュアル公開」と同時に行い、インパクトを最大化する
- 公式サイトOPENをタイトル解禁と連動させ、情報の受け皿を先に作っておく

### フェーズ4：DEMO体験版公開

#### 設計方針
- DEMO版は「ゲームへの入口」。プレイした人が自然と感想を投稿したくなる仕掛けを設計する
- 公式側からの感想収集投稿を意図的に出し、UGC（ユーザー生成コンテンツ）の波紋を作る
- DEMO版はあくまで「体験版」であることを明示し、正式版への期待を高めることを忘れない
