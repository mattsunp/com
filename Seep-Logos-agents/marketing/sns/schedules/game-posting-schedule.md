# ゲーム SNS投稿スケジュール（全フェーズ版）

> 対象：新作ゲーム（コンシューマー・スマホアプリ両対応）の正式発表以降から運用終了まで
> sns-manager作成｜ブランドルール適用済み
>
> **注意：発表前ティザー期〜正式発表当日のスケジュールは `announcement-schedule.md` を参照すること。**
> **本資料はその後続、すなわち「正式発表以降〜リリース後の継続運用フェーズ」を扱う。**

---

## 全体タイムライン

```
【フェーズ0】発表前期（ティザー）
  → announcement-schedule.md を参照

【フェーズ1】発表後〜発売3ヶ月前
  情報開拓期：週2〜3回。キャラ・世界観・システムの核心を届ける

【フェーズ2】発売3ヶ月前〜発売1ヶ月前
  加速期：週3〜5回。事前登録・体験版・メディア露出と連動

【フェーズ3】発売1ヶ月前〜発売当日
  クライマックス期：週5回〜毎日。カウントダウン・最終解禁

【フェーズ4】発売後（継続運用期）
  定常運用期：週3〜5回。アップデート・イベント・コミュニティ維持
```

---

## フェーズ1｜発表後〜発売3ヶ月前（情報開拓期）

### フェーズ概要

| 項目 | 内容 |
|---|---|
| 投稿頻度 | 週2〜3回（過剰投稿によるフォロワー離脱を防ぐ） |
| 主な投稿時間 | 20:00〜22:00（ゴールデンタイム優先） |
| サブ投稿時間 | 12:00〜13:00（昼休み帯） |
| 推奨曜日 | 水曜・木曜・日曜（エンゲージメントが安定する傾向） |

### 週次ローテーション（コンテンツ種別）

| 週 | 投稿内容 | メディア |
|---|---|---|
| 第1週 | キャラクター第1弾詳細設定 | キャラクターシート or 立ち絵 |
| 第2週 | ゲームシステム紹介第1弾 | スクリーンショット or 説明図 |
| 第3週 | キャラクター第2弾 | 上記に準ずる |
| 第4週 | 世界観・ロア（世界の成り立ち・用語）公開 | コンセプトアート or 設定資料 |
| 第5週以降 | キャラ・システム・世界観をローテーションして継続 | — |

### コンテンツ設計の指針

- **キャラクター投稿**：プロフィール＋「好きなもの/信条/一言」等の断片情報を添える。ファンが二次創作・考察したくなる余白を残す
- **ゲームシステム投稿**：「なぜこのシステムが面白いか」という体験の核心を1投稿1トピックで届ける
- **世界観投稿**：設定の全量ではなく「世界の断片」を届けることで想像力を刺激する。用語・固有名詞は最小限に絞る

## フェーズ2｜発売3ヶ月前〜発売1ヶ月前（加速期）

### フェーズ概要

| 項目 | 内容 |
|---|---|
| 投稿頻度 | 週3〜5回 |
| 投稿時間 | 20:00〜22:00 優先、必要に応じて18:00〜20:00 も活用 |
| 推奨曜日 | 月・水・金・日（週4回体制の場合） |

### このフェーズで発信すべき重点コンテンツ

| 施策 | タイミング | 内容 |
|---|---|---|
| 事前登録開始告知 | フェーズ開始直後（発売90日前前後） | 事前登録URL・特典内容を明確に告知 |
| 事前登録マイルストーン報告 | 10万・30万・50万人等の節目 | 達成感謝＋次の目標を共有 |
| PV第2弾公開 | 発売70日前前後 | ゲームプレイの実際を見せる映像 |
| 体験版・クローズドβ告知 | 発売60日前前後 | 参加方法・期間・対応機種を明示 |
| メディアレビュー解禁告知 | レビュー記事公開と同時 | 掲載メディア名を記載して信頼感を添える |
| スタッフインタビュー・開発秘話 | 月1回程度 | 「作り手の顔」を見せてブランドへの共感を深める |
| コラボ・タイアップ発表 | 確定次第 | 他IPや著名クリエイターとの連携 |

### 事前登録キャンペーンの投稿設計

```
【告知投稿のフォーマット】
本文：
  - 事前登録開始宣言（簡潔に1〜2文）
  - 特典の具体的提示（キャラクター・アイテム名を明記）
  - 事前登録URL（短縮URL推奨）
ハッシュタグ：公式タグ＋ゲームジャンルタグ＋事前登録タグ
メディア：特典ビジュアル
```

## フェーズ3｜発売1ヶ月前〜発売当日（クライマックス期）

### フェーズ概要

| 項目 | 内容 |
|---|---|
| 投稿頻度 | 週5回〜毎日 |
| 投稿時間 | 20:00〜22:00（最高優先） + 朝8:00（宣言投稿） |

### カウントダウン投稿設計

| タイミング | 内容 |
|---|---|
| 発売30日前 | 「あと1ヶ月」告知＋総まとめビジュアル |
| 発売14日前 | 「あと2週間」＋未公開情報の小出し解禁 |
| 発売7日前 | 「あと1週間」＋最終PVまたはローンチトレーラー公開 |
| 発売3日前 | 「あと3日」＋予約・購入導線の最終強調 |
| 発売前日 | 「明日リリース」＋DL方法・起動時刻の案内 |
| 発売当日 | 朝8:00「本日リリース」宣言 → 0:00/9:00ストア公開と連動投稿 |

### 発売当日の投稿時刻設計

| 時刻 | 内容 |
|---|---|
| 0:00（深夜0時リリースの場合） | ダウンロード開始宣言・ストアリンク |
| 8:00〜9:00 | 起床層への告知「本日リリースです」 |
| 12:00 | 昼休み層へのリマインド＋インフルエンサー実況誘導 |
| 20:00〜21:00 | 夜間層への感謝投稿＋初日プレイ感想拡散 |

## フェーズ4｜発売後（継続運用期）

### 概要

| 項目 | 内容 |
|---|---|
| 投稿頻度 | 週3〜5回（安定期は週3回を基本とし、イベント時に増加） |
| 基本投稿時間 | 20:00〜22:00（コンシューマー） / 12:00・20:00（スマホアプリ） |

### コンテンツローテーション（週次テンプレート）

| 曜日例 | コンテンツ種別 | 目的 |
|---|---|---|
| 月曜 | アップデート・メンテナンス告知 | 情報提供・不満先取り |
| 水曜 | ゲーム内イベント情報・新コンテンツ紹介 | 継続プレイの動機付け |
| 金曜 | ユーザー投稿紹介（UGC活用）or キャンペーン告知 | コミュニティ活性化 |
| 日曜 | 週末ログイン促進 or 実績・記念日投稿 | エンゲージメント維持 |

### 継続運用で意識すべきコンテンツ設計

#### 定例コンテンツ（固定化して安心感を作る）
- **メンテナンス予告・完了報告**：プレイヤーへの配慮を示す基本投稿
- **イベント開始・終了告知**：開始3日前・前日・当日の3点セットが基本
- **バージョンアップ情報**：新機能・新キャラ・バランス調整の要点を端的に

#### 波紋型コンテンツ（拡散を狙う非定例）
- **累計DL・プレイヤー数節目投稿**：「100万DL達成」等。感謝＋記念キャンペーンとセットにすると効果的
- **周年記念投稿**：ファンアートコンテスト・特別ライブ配信・限定グッズ告知と連動
- **コラボ発表**：IP・アーティスト・インフルエンサーとの連携。エンゲージメント最大化施策の核

#### ファン参加型コンテンツ
- **ファンアート紹介**：公式が取り上げることでUGC（ユーザー生成コンテンツ）創作意欲を高める
- **#実況OK・生放送歓迎タグ**：配信者コミュニティとの接点を作る
- **アンケート機能活用**：「あなたの推しキャラは？」等、ファンが気軽に参加できる仕掛け

## 投稿頻度のフェーズ別まとめ

| フェーズ | 期間目安 | X(Twitter) | Instagram | YouTube |
|---|---|---|---|---|
| 情報開拓期 | 発表後〜3ヶ月前 | 週2〜3回 | 週1〜2回 | 月2〜3回（PV・設定動画） |
| 加速期 | 3ヶ月前〜1ヶ月前 | 週3〜5回 | 週2〜3回 | 月3〜4回 |
| クライマックス期 | 1ヶ月前〜発売当日 | 毎日〜週5回 | 週3〜4回 | 週1〜2回（トレーラー・カウントダウン） |
| 継続運用期 | 発売後〜 | 週3〜5回 | 週2〜3回 | 月2〜4回（アップデート解説・イベント動画） |

## ゲーム種別による差分

| 観点 | コンシューマーゲーム | スマホアプリゲーム |
|---|---|---|
| 発表〜発売の期間 | 6ヶ月〜1年以上 | 事前登録から平均60日前後 |
| 最重要フェーズ | 発売3ヶ月前〜当日 | 事前登録開始〜リリース当日 |
| 継続運用の軸 | DLC・アップデート・周年 | 定期イベント・ガチャ更新・コラボ |
| 投稿のトーン | 世界観・没入感重視 | 速報性・特典訴求重視 |
| 推奨投稿時間（継続） | 20:00〜22:00 | 12:00・20:00〜22:00の2段構え |

---

## 投稿時間帯の優先度（フェーズ共通）

| 優先度 | 時間帯 | 理由 |
|---|---|---|
| 最高 | 20:00〜22:00 | ゲーム・エンタメ層の活動ピーク。帰宅後リラックスモード |
| 高 | 18:00〜20:00 | 帰宅途中〜夕食時 |
| 中 | 12:00〜13:00 | 昼休み帯 |
| 補助 | 8:00〜9:00 | 発売当日・重大発表時の宣言投稿のみ |
| 避ける | 1:00〜7:00 | 深夜〜早朝（リーチが著しく低下） |

## 炎上・品質リスクチェックリスト（投稿前確認）

- [ ] 未発表情報・ネタバレが含まれていないか
- [ ] メンテナンス・障害情報と混同されるトーンになっていないか
- [ ] ユーザーの不満に対する配慮を欠いた内容になっていないか
- [ ] コンシューマー版・スマホ版で内容が混在していないか
- [ ] 競合他社の名指し・過度な比較が含まれていないか
- [ ] 「業界初」「唯一無二」等の根拠なき誇大表現がないか
