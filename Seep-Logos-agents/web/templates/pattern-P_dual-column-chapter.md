# ワイヤーフレーム：Pattern P「Dual Column Chapter型」

作成日：2026-03-31
担当エージェント：デジタル・技術部 / web-director
ビジュアルレビュー：アートディレクション部 / art-director

## 概要
- **用途**：世界観・物語・キャラクター紹介を、画像とテキストの対話で見せたい作品。ゲームIP、アニメIP、ノベルIPの特設ページ。「読む」体験より「歩く・発見する」体験を優先するとき
- **ページ数**：1ページ（縦スクロール・チャプター交互展開型）
- **必要最低素材**：タイトルテキスト、各チャプターのテキスト（100〜300字 × 3〜6章）、ブランドカラー（1〜2色）。画像は任意（プレースホルダーで完全代替可）
- **推奨スタイル**：ディープダーク背景、センターラインに発光色（アンバー・シアン・白など）、モノスペース体 × セリフ体の混合、左右交互の余白リズム

---

## 差別化ポイント（対 A〜O）

| 軸 | 本パターンの立場 |
|---|---|
| vs Pattern N（Chapter Scroll） | Nは縦1カラムで「書物を読む」体験。Pは左右2カラム交互で「展覧会を歩く」体験。画像スロットがある点も差異 |
| vs Pattern E（Scrollytelling） | Eはスクロール演出（視差・背景）が主役。Pはレイアウト構造そのものが没入の主軸。演出は補助 |
| vs Pattern M（Timeline Immersion） | Mは横スクロール×時系列。Pは縦スクロール×左右交互。時間軸ではなく「発見の軸」で章を並べる |
| vs Pattern G（Character Encyclopedia） | Gはキャラクター図鑑（グリッド・カード型）。Pはチャプター進行型で物語文脈の中でキャラや場面を見せる |
| 独自性 | 中央センターライン＋左右交互2カラム＋スクロール連動描画演出の組み合わせは A〜O に存在しない |

---

## センターライン設計方針（アート管理者補足）

センターラインは装飾ではなく「物語の背骨」として機能させる。

- **色**：アンバー（#d4a843）推奨。作品のブランドカラーに差し替え可
- **太さ**：2px。細すぎず太すぎず、存在を主張しながら読書を邪魔しない
- **描画演出**：スクロール量に応じて上から下へ線が伸びる。`scaleY()`トランスフォームとスクロールイベントで連動。IntersectionObserverと組み合わせてチャプター到達時に加速する
- **グロウ**：`box-shadow`で縦方向の光の滲みを付与。線が光を帯びているように見せる
- **接続点**：各チャプターの区切りで、センターラインから左右へ短い横線（接続ノード）が伸びる。ラインが「章を束ねている」構造を視覚化する

## ページ構成

```
┌─────────────────────────────────────────────────────────────────┐
│  [CHAPTER NAV]（固定・右端・縦型インジケーター）                │
├─────────────────────────────────────────────────────────────────┤
│  [HERO] タイトル・キャッチコピー・スクロール導線                │
├──────────────────────┬──────────────────────────────────────────┤
│         [CHAPTER 1]  │                                          │
│  左：画像プレース    │  右：テキスト                            │
│  ホルダー            │  （章タイトル＋本文＋引用）              │
│                      │                                          │
├──────────────────────┴───────────────────────────┬──────────────┤
│                              [CHAPTER 2]         │              │
│  左：テキスト                                    │ 右：画像     │
│  （章タイトル＋本文＋引用）                      │ プレース     │
│                                                  │ ホルダー     │
├──────────────────────┬───────────────────────────┴──────────────┤
│         [CHAPTER 3]  │                                          │
│  左：画像プレース    │  右：テキスト                            │
│  ホルダー            │                                          │
├──────────────────────┴──────────────────────────────────────────┤
│  [EPILOGUE / CTA] 締めの言葉＋登録・購入                        │
├─────────────────────────────────────────────────────────────────┤
│  [FOOTER]                                                       │
└─────────────────────────────────────────────────────────────────┘
```

中央の縦線（センターライン）はすべてのセクションを貫通し、スクロールに連動して上から「描かれていく」。

## CHAPTER NAV（固定・右端・縦型）

```
              ┌────┐
              │  ○ │ ← HERO（通過済み：塗りつぶし）
              │  │ │
              │  ○ │ ← CHAPTER 1（読中：グロウパルス）
              │  │ │
              │  ○ │ ← CHAPTER 2
              │  │ │
              │  ○ │ ← CHAPTER 3
              │  │ │
              │  ○ │ ← EPILOGUE
              └────┘
```

**要素メモ：**
- 右端固定・幅20px、各チャプターをドット（○）で表示
- スクロール位置に応じてアクティブドットがハイライト（グロウ＋アンバー）
- 通過済みドットは塗りつぶし、未到達は枠線のみ
- ドットクリックで該当チャプターへスムーズスクロール
- ホバー時にチャプタータイトルがツールチップ表示
- モバイルでは非表示

```
┌─────────────────────────────────────────┐
│                                         │
│  　　　　　　　　　　　　　　　　　　   │
│      [PLACEHOLDER: タイトルロゴ]         │
│      ※ なし時：大きなタイトルテキスト   │
│      （セリフ体・中央揃え）              │
│                                         │
│      ─────────  │  ─────────            │
│                  │（センターラインの始点）│
│                                         │
│      [PLACEHOLDER: キャッチコピー 1〜2行]│
│      例：「二つの視点が、ひとつの真実へ」│
│                                         │
│                  │                      │
│              ▼  続きを見る              │
│                                         │

```

**アニメーション：**
- タイトルがフェードイン（0.6s）

- センターラインの起点が点灯する（パルスエフェクト）
- 「▼ 続きを見る」が上下パルス

## CHAPTER（各章・min-height: 100vh）

### 奇数チャプター（1, 3, 5...）：左＝画像、右＝テキスト

```
┌──────────────────────┬──│──┬──────────────────────┐
│                      │  │  │                      │
│  ┌──────────────────┐│  │  │  CHAPTER  I          │
│  │                  ││  │  │  ─────────────────    │
│  │  [PLACEHOLDER:   ││  │  │  [章タイトル]         │
│  │   画像エリア]    ││  │  │  例：「出会いの地」   │
│  │                  ││  │  │                      │
│  │  ※ なし時：      ││  │  │  [本文テキスト       │
│  │  ジオメトリック  ││  │  │   100〜300字]        │
│  │  プレースホルダー││  │  │                      │
│  │  （アスペクト比  ││  │  │  ── 引用テキスト ──  │
│  │   4:3 推奨）     ││  │  │  「──象徴的な一文」  │
│  │                  ││  │  │                      │
│  └──────────────────┘│  │  │                      │
│                      │  │  │                      │
└──────────────────────┴──│──┴──────────────────────┘
                           │
                  （センターライン継続）
```

### 偶数チャプター（2, 4, 6...）：左＝テキスト、右＝画像

```
┌──────────────────────┬──│──┬──────────────────────┐
│                      │  │  │                      │
│  CHAPTER  II         │  │  │  ┌──────────────────┐│
│  ─────────────────   │  │  │  │                  ││
│  [章タイトル]        │  │  │  │  [PLACEHOLDER:   ││
│                      │  │  │  │   画像エリア]    ││
│  [本文テキスト]      │  │  │  │                  ││
│                      │  │  │  │  ※ なし時：      ││
│  ── 引用テキスト ──  │  │  │  │  ジオメトリック  ││
│  「──象徴的な一文」  │  │  │  │  プレースホルダー││
│                      │  │  │  └──────────────────┘│
│                      │  │  │                      │
└──────────────────────┴──│──┴──────────────────────┘
                           │
                  （センターライン継続）
```

**接続ノード（センターラインから水平に伸びる短線）：**
- 各チャプターの開始点でセンターラインから左右へ24px伸びる横線が出現
- この横線が「章の始まり」を示す視覚的マーカーとなる
- チャプターがビューポートに入ったとき、横線が外側に向かってアニメーションで伸びる

**各チャプター入場アニメーション（IntersectionObserver使用）：**
- 画像エリア：外側（左または右）からスライドイン（0.5s）
- テキストブロック：内側（センターライン側）からフェードイン（0.3s delay）
- 引用テキスト：本文より遅れて表示（0.5s delay）、フォントサイズ1.3倍

## センターライン実装仕様

```
position: fixed または sticky（HEROからFOOTERまで貫通）
width: 2px
left: 50%
top: 0
background: linear-gradient(to bottom, transparent, #d4a843 5%, #d4a843 95%, transparent)
box-shadow: 0 0 8px rgba(212, 168, 67, 0.6), 0 0 20px rgba(212, 168, 67, 0.2)
transform-origin: top center
transform: scaleY(0) → scaleY(1)（スクロール量に比例）
```

**スクロール連動ロジック：**
```
scrollProgress = scrollY / (documentHeight - viewportHeight)
centerLine.style.transform = `scaleY(${scrollProgress})`
```

チャプター到達イベントでグロウを強調する（box-shadowを一時的に拡大）。

## プレースホルダー一覧

| 箇所 | 内容 | 素材なし時の代替 |
|---|---|---|
| タイトルロゴ | SVGロゴ・画像 | タイトルテキスト（セリフ体・大サイズ） |
| キャッチコピー | 1〜2行 | 必須（作品のキャッチコピー） |
| 章タイトル | 各章の名称 | 必須（例：「第一章　出会いの地」） |
| 本文テキスト | 100〜300字 | 必須（これが主役） |
| 引用テキスト | 章の核心を抜き出した1文 | 任意（なくても成立） |
| 画像エリア（各章） | コンセプトアート・キャラクター・場面イラスト | アスペクト比4:3のジオメトリックプレースホルダー（CSS生成） |
| 締めの言葉 | EPILOGUE締め文1〜3文 | 必須 |
| 背景 | なし（CSSで代替） | 不要 |

## タイポグラフィ方針

| 用途 | フォント種別 | サイズ目安 |
|---|---|---|
| タイトル（HERO） | セリフ体（Georgia / Noto Serif JP） | 3.5〜5rem |
| CHAPTER番号 | モノスペース / サンセリフ（細） | 0.7rem、大文字・tracking広め |
| 章タイトル | セリフ体 | 1.6〜2.2rem |
| 本文テキスト | セリフ体または読みやすいサンセリフ | 1rem〜1.1rem、行間2.0 |
| 引用テキスト | セリフ体・斜体風 | 1.3〜1.5rem、中央揃えまたは外側揃え |
| CTAボタン | サンセリフ（細） | 0.875rem |

**アート管理者補足：**
左カラムと右カラムのテキストは同じ書体でも「揃え方向」で差をつけること。
奇数章（右テキスト）は左揃え、偶数章（左テキスト）は右揃えを基本とする。
これにより、センターラインを「鏡」として左右のテキストが向き合う構造が生まれる。
引用テキストは必ず水平方向に余白（padding）を広くとり、「息を吸う空間」を確保する。

## 拡張ロードマップ

1. チャプター数は3〜6で調整可。偶数が推奨（左右対称で終わるリズム感）
2. 画像プレースホルダーに動画（ループ短尺）を差し込むとより映像的になる
3. センターラインの色を章ごとに変化させると、物語の感情曲線を色で表現できる
4. 各チャプターにBGMを割り当て、センターラインが章に到達したとき音楽が変わる演出
5. 縦書きバリアント：テキスト側を`writing-mode: vertical-rl`で縦書きにすると、横向きの画像×縦向きのテキストというコントラストが強まる
6. 画像解禁後：プレースホルダーをそのままキーアート・場面絵に差し替え可能
