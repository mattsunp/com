# ワイヤーフレーム：Pattern E「Scrollytelling型」

作成日：2026-03-30
担当エージェント：デジタル・技術部 / web-director

## 概要
- **用途**：スクロールで世界観を「体験」させる。ナラティブ・ストーリー主軸の作品向き
- **ページ数**：1ページ（縦スクロール・チャプター構造）
- **必要最低素材**：ロゴ、世界観テキスト（チャプター×3〜5）、ブランドカラー
- **推奨スタイル**：スクロール連動アニメーション、視差効果（Parallax）、GSAP等使用

---

## NAV（初期表示のみ・スクロール開始で消える）

```
┌─────────────────────────────────────────┐
│ [ロゴ/タイトル]                  [登録] │
└─────────────────────────────────────────┘
```

**要素メモ：**
- スクロール開始と同時にフェードアウト
- ENDINGセクションで再びフェードイン

## SECTION 2〜4：CHAPTER（各100〜150vh）

各チャプターは同じ構造を繰り返す。スクロール量に応じてテキスト・ビジュアルが出現。

```
┌─────────────────────────────────────────┐
│                                         │
│  ── C H A P T E R  0 1 ──              │
│  [PLACEHOLDER: チャプタータイトル]       │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │                                 │    │
│  │  [PLACEHOLDER: 横幅広めビジュアル│    │
│  │   コンセプトアート・背景・       │    │
│  │   スクリーンショット等]          │    │
│  │                                 │    │
│  └─────────────────────────────────┘    │
│                                         │
│  [PLACEHOLDER: 世界観テキスト 100〜200字]│
│  ※ スクロール位置に達したらフェードイン │
│                                         │
│  [PLACEHOLDER: キーワードタグ ×2〜3]    │
│                                         │
└─────────────────────────────────────────┘
```

**チャプター構成例（3チャプター）：**
| チャプター | テーマ例 | ビジュアル |
|---|---|---|
| 01 | 世界観・舞台設定 | コンセプトアート・背景 |
| 02 | 主人公・キャラクター | キャラクターアート |
| 03 | 事件・物語の発端 | キーシーン・象徴的ビジュアル |

**アニメーション例：**
- テキストが下から滑り上がってくる（slide-up）
- ビジュアルが奥から手前へズームイン
- 背景要素が異なる速度で流れる（多層視差）
- ビジュアルなし時：テキストのみを大きく、行間広めに配置

## プレースホルダー一覧

| 箇所 | 内容 | 素材なし時の代替 |
|---|---|---|
| OPENING背景 | キーアート・世界観ビジュアル | 深色グラデーション |
| タイトルロゴ | SVGロゴ | タイトルテキスト |
| CHAPTERビジュアル | コンセプトアート・背景・キャラ | テキストのみ（大きめフォント） |
| 世界観テキスト | 各章100〜200字 × 3〜5章 | 必須（これが主役） |
| キャッチコピー | 1〜2行 | 必須 |

---

## 拡張ロードマップ

1. チャプター数は3〜5で調整可。素材が揃うにつれ追加
2. 各チャプターにBGM・SEを追加するとさらに没入感が上がる
3. 情報解禁後：CHAPTERをSTORY・CHARACTERS・SYSTEMの独立ページに分割してPattern Jへ移行
