# 企画書

## MNEMOSYNE — 他者の記憶を消す仕事をしてきた人間が、自分の消された記憶に気づく

### コアコンセプト
他者の記憶を消すことを仕事にしてきた人間が、自分の消された記憶に気づく。
90〜120分。答えは出ない。しかし問いが残る。

### ジャンル
一人称ナラティブゲーム（短編アドベンチャー）

### ターゲット
- 主軸：20〜35歳、文学・映画・インディーゲームに親しむ層（「IMMORTALITY」「What Remains of Edith Finch」「BEFORE YOUR EYES」等の体験者）
- 副軸：記憶・喪失・アイデンティティをテーマにした作品に関心を持つ層

### ゲームプレイの核心方針
選択はプレイヤーの実感を深めるためにのみ存在する。そのため選択の数は少なく、しかし重い。

### 競合との差別化
「記憶の整理」という職業的行為をゲームプレイの中心に置く。「他者の記憶を操作する職業人」を主人公にすることで、プレイヤーが倫理的問いに自然に引き込まれる。単純な観察ゲームではなく、プレイヤーの分類行動がセツ自身の記憶に影響を与えるフィードバック構造を持つ。

### メディア展開計画
1st：短編ゲーム（Steam/itch.io） → 2nd：短編小説・ノベル展開 → 3rd：フルプライス版または続編

### 収益モデル
短編有料DL（1,000〜1,500円）。クリエイティブスタジオの実績作品・名刺代わりのポジショニング。

### タイトルについての備考
MNEMOSYNE（ムネモシュネ）はギリシャ神話の記憶の女神に由来する固有名詞。国際展開時にローカライズが困難になるリスクあり。itch.io等での初期展開では固有名詞タイトルが記憶性・検索性を高めるため現タイトルを維持する。続編・海外展開時に副題またはサブタイトルで補完することを検討。

# 世界観設定

## 世界の基本原則

2040年代の日本。**記憶の外在化技術（Memory Externalization Technology、MET）** が実用化されて15年が経過している。

**技術的ルール：**
- 人間の記憶は脳神経活動として計測・データ化が可能になった
- データ化された記憶は「体験の再現」ではなく「統計的な近似」である——この不可逆性が世界の根本的矛盾
- 記憶の消去は完全消去ではなく「アクセス封鎖」であり、感情強度が高い記憶ほどアクセス封鎖が不完全になりやすく、リークが生じやすい

**記憶消去の申請権限（確定設定）：**
原則は自己申請のみ。例外として以下が認められる：
- 未成年者：保護者申請可能
- 判断能力が著しく低下した場合：後見人申請可能
- 犯罪被害者支援：「証拠記憶保護令状」による強制保全（被害者本人の申請なしに被害記録を保全する制度）

→「第三者が他者の記憶を消すことはできない」が原則。ただし本作の物語では「どうやって消したか」の謎が後半で問われる。

**社会的状況：**
- 記憶消去サービスは医療・労務・法曹等で制度化。トラウマ治療から「不採算な記憶の整理」まで幅広く利用される
- 記憶を「保有することのコスト」を語る社会言説が定まりつつある
- **メモリスト（記憶原理主義者）** が存在し、記憶消去を人格の侵害として抵抗運動を行っている。本作の短編スコープでは背景ノイズとして示唆する程度（街頭の抗議活動、同僚の呟き）。続編・フルプライス版での展開軸として保留。

## 記憶整理士という職業

**正式名称：認定記憶整理士（Licensed Memory Coordinator）**

- 依頼者の記憶データを精査し「保持すべき記憶」「封鎖すべき記憶」「デジタルアーカイブすべき記憶」を分類・提案する専門職
- 倫理委員会の認定資格が必要
- 最大の職業的ジレンマ：「依頼者が忘れたがっている記憶」と「社会的に保全すべき証拠記憶」が衝突するケース

## リーク現象

**専門用語：リーク（Leak）**

封鎖された記憶が夢・既視感・感情反応として断片的に浮上する現象。感情強度が高い記憶ほど封鎖が不完全になりやすい。本作においてこの設定は物語的必然として機能する：セツの消された記憶がリークするという事実は、「その記憶が強い感情を帯びていた」ことの間接証拠になる。

## 用語集
- **MET（Memory Externalization Technology）**：記憶外在化技術
- **アクセス封鎖**：記憶消去の実態。完全削除ではなくアクセスを遮断する処理
- **リーク（Leak）**：封鎖された記憶が断片的に浮上する現象
- **認定記憶整理士（LMC）**：記憶整理の専門資格を持つ職業人
- **メモリスト**：記憶消去に反対する原理主義者の総称

## 設定未確定リスト
- MET技術の開発経緯・開発元企業の詳細
- メモリスト組織の規模・活動実態（続編で解像度を上げる）
- 「消去の痕跡」が法的証拠として扱われるかどうか

# シナリオ骨格

## 核心となる問い
忘れることは、罪か。

主人公が他者の記憶を消去してきた10年の仕事は、「癒し」だったのか「抹消」だったのか。その問いが、自分の消された記憶を辿る旅の中で、答えを変え続ける。

## 主人公：桐島セツ（33歳）

認定記憶整理士。10年のキャリア。

**失っているもの：** 自分の過去に起きたある出来事の記憶。本人が依頼して消したのか、他者が消したのかは不明。
**求めているもの：** 表向きは「何が消えたかを知ること」。本当は「消えた記憶を持っていた自分が正しかった、という証明」。
**恐れているもの：** 「消えた記憶が、消えるべきだったもの」という可能性。

## 登場する依頼者

**依頼者A（40代・女性）：**
夫との離婚に関わる記憶の整理を依頼。「愛した記憶」と「傷ついた記憶」を分離できないことに苦しんでいる。セツの物語の鏡像として機能する。

**依頼者B（60代・男性）：**
消去済みの記憶が強烈にリークしている状態で来訪。リーク記憶の中の人物がセツ自身の消えた記憶と重なる断片を持っている。物語の転換点を担う。

## 「消えかけた存在」について（確定設定）

本作における「消えた記憶の中の存在」は語りかけない。言葉ではなく記憶の断片として現れる。プレイヤーは「これは人物の残影か、それとも自分の感情の投影か」を問い続けるが、ゲームは答えを提供しない。これはテーマに対して誠実な設計判断である。

## 三幕構造

**第一幕：職業という防壁**
セツは他人の記憶整理を完璧にこなす。依頼者Aのケースに取り組む。依頼者の痛みに触れながらも感情を切り離している。依頼者Aの封鎖記憶がリークする現場に立ち会うとき、自分の記憶にも同様の「痕跡」があることに気づく。

**第二幕：痕跡を辿る**
セツは自分の記憶ログを遡る。依頼者Bのケースが転換点となり、断片的なリーク映像から「消された人物」の輪郭が浮かぶ。辿るほどに、セツが信じていた「自分の職業倫理」が揺らぐ。

**第三幕：選択の前に立つ**
「消された記憶」の全容に手が届く直前で、選択が訪れる。直前にプレイ振り返りサマリが挿入される（「あなたはここまでの仕事で何を選んできたか」）。選択肢：
- 思い出す（リーク封鎖を解除する）
- このまま忘れていく（封鎖を維持する）
- 記録だけを残して、自分の感情的なアクセスを永久に封じる（第三の道）

「全容を知る前に選ばされる」構造は意図的な設計である：セツが10年かけて依頼者に行ってきたことと同じ構造——「全容を知らないまま消去を勧める」——をプレイヤー自身が経験する。これはテーマの体現。

## 感情動線（確定版）
好奇心 → 不安 → 怒り（「誰が消した？なぜ消した？」）→ 否認（「消えた記憶など最初からなかったかもしれない」）→ 共鳴（依頼者Aの痛みと自分の痛みが重なる瞬間）→ 罪悪感 → 静けさ（どの選択をしても）

## エンディング設計

3種のエンディングそれぞれに、プレイヤーが最も多く選んだ分類カテゴリ（整理・保持・封鎖）によって変わる1〜2行のセツの独白を追加する。

- 「保持」を最も多く選んだプレイヤーのセツ：忘れることを恐れる人間として語る
- 「封鎖」を最も多く選んだプレイヤーのセツ：選択の重さを知りながらも前を向く人間として語る
- 「整理」を最も多く選んだプレイヤーのセツ：感情より機能を優先してきた職業人として語る

# ゲームプレイ設計

## ゲームループ

### 短期ループ（1アクション単位）
「フラグメントを見る → 感情強度を感じる → 整理・保持・封鎖を選ぶ」

プレイヤーは記憶の断片を受け取り、3カテゴリに分類する。「正解はない、しかし選択には重さがある」。感情的に重い選択をするほどセツのリーク解放速度が変化する。選択のたびにセツの独白トーンが微細に変化する。

### 中期ループ（1セッション単位）
依頼者Aのケース → セツのリーク解放（1層目） → 依頼者Bのケース → セツのリーク解放（転換点）→ 第三幕へ

依頼者のケースを処理するたびにセツの記憶が解放されるゲートが開く。

### 長期ループ（クリアまで）
「消された記憶の全容に向かう → プレイ振り返りサマリ → 三択選択 → 個人化された独白でエンディング」

## プレイヤー動機

**内発的動機：**
- 「消えた記憶は何か」という謎解きの好奇心
- 自分が下した分類判断への罪悪感・責任感（他者の大切な記憶を消してしまったかもしれない）
- 依頼者の人生の断片に触れることによる共感

**外発的動機：**
- 依頼者ケース完了によるセツの記憶フラグメント解放
- 隠し記憶（整理の中でたまたま発見するもの）による探索欲

## 感情表現の設計

セツの感情的状態はUI数値化しない。以下の環境変化として表現する：
1. **セツの独白テキストのトーン変化**：「職業的距離（冷静・専門的）」→「揺れ始め（自問が混入）」→「沈黙の近く（断言が消え問いだけ残る）」
2. **BGM・環境音の変化**（アートディレクターと要確認）
3. **画面の色温度変化**（アートディレクターと要確認）

これにより世界観の没入を維持しながら感情の変化をプレイヤーに伝達する。

## 難易度曲線

- **序盤**：依頼者Aのケースを通じてシステムを学習。チュートリアルを職場の日常として自然に組み込む。
- **中盤**：自分の記憶断片が混入し始め、「どれが誰の記憶か」の区別が曖昧になる。依頼者Bのケースで認知負荷がピークに達する。
- **終盤**：セツの記憶と依頼者の記憶が重なり合い、分類の判断が感情的に重くなる。

## 未解決・要確認事項
- 「記憶フラグメント」の表現形式：テキスト・映像・音声のどれか（アートディレクターと要確認）
- プレイ時間90〜120分の密度設計：依頼者2名のケースとセツの謎の配分比率
- 分類操作のUI設計：worldbuilding-creatorの没入原則とgame-directorの操作フィードバック設計の最終調整（game-ui-designerに委ねる段階）
