# 短編ゲーム企画：『Yesterday's Room』（v6最終版）

*生成日：2026-03-28 | Seep Logos クリエイティブスタジオ*
*オーケストレーション：全体戦略家（chief-producer）v6実施*
*参加エージェント：企画人・世界観構築人・シナリオ書き（Round 1〜3 + 相互批評）*

---

## v6の設計方針（全体戦略家より）

v5（PALIMPSEST）は「他者の記憶を読む」設計で完成した。
v6は意図的に反転させる。「自分の記憶が消えていく」を主軸に置く。

制約：
- ジャンルはパズルナラティブ以外
- 世界のスケールは「一つの部屋（群）」に限定
- 近未来テクノロジーを使わない

---

## コンセプト（最終）

**タイトル**：『Yesterday's Room』（暫定）
**サブタイトル候補**：― 忘れていく、それでも ―
**ジャンル**：探索アドベンチャー × フラッシュバック選択型
**プレイ時間**：30〜45分（要件内）
**プレイ人数**：1人
**推奨開発エンジン**：Unity（URP）または Godot 4

**ターゲット**：
- メイン：25〜45歳、ストーリー重視のゲーマー。GONE HOME / What Remains of Edith Finch 系統の愛好者
- サブ：介護・老齢の親を持つ世代（非ゲーマー含む）。感情的接続が強い層

**コアコンセプト**：
「忘れることを、身体で体験する」

プレイヤーは「記憶が減っていくこと」を情報として受け取るのではなく、
**操作の手応えが少しずつ消えていくこと**で体感する。

**コアループ**：
```
部屋を歩く → 物を見る（ラベルの変化に気づく）
    → キーオブジェクトでフラッシュバック発動
    → フラッシュバック内で選択
    → 「今の武」に何かが残る、または消える
    → 別の部屋へ
    → 前の部屋に戻ると、ラベルや状態が変化している
```

## 世界観（最終）

**舞台**：現代日本・地方都市の一軒家。2010年代後半。
**空間**：居間・台所・書斎・寝室・縁側・玄関の6空間。

**主人公**：宮本 武（みやもと たける）、82歳。元高校美術教師。
妻・良子（よしこ）に3年前に先立たれた。
娘・恵（めぐみ）は遠方に嫁ぎ、年に数回来る。

**世界観の核心**：手書きラベルのある家

武は認知症の初期段階にある。主治医の勧めで、家じゅうの物に「手書きのラベル」を貼っている。
これはゲームのビジュアル言語であり、忘却の進行を示す指標となる。

```
序盤のラベル：
  冷蔵庫 → 「冷蔵庫。右上に牛乳。卵は12個入り。」
  電話  → 「電話。娘の番号は 042-xxx-xxxx。」
  写真立て → 「良子と武。結婚式。1975年。」

中盤（別の部屋から戻ったとき）：
  冷蔵庫 → 「冷蔵庫。しろいもの。」
  電話  → 「電話。むすめ。」
  写真立て → 「女の人と男の人。」

終盤（書斎に戻ったとき）：
  棚のスケッチブック → 「…」
  油絵  → （ラベルなし）
```

**例外：ラベルのない物**

台所の赤いヤカンにだけ、ラベルが貼られていない。
武はそのヤカンの前で何度もペンを持ったが、貼れなかった。
「ラベルを貼ることが忘れに抗う行為」だとすれば、
貼れなかったヤカンは「抗えなかったもの」の象徴だ。

**美術教師という職業の意味**：
武は「見ること」を職業にした人間だ。
認知症で最初に失われるのは「名前」だが、武は最後まで「形と色」を失わない。
ゲームのビジュアルは最後まで美しいまま。
ただし「名前ラベル」だけが壊れていく。

**場所の象徴設計**：

| 場所 | 象徴 |
|---|---|
| 書斎 | 若い頃の自分（教師・創造者） |
| 台所 | 良子との日常（失われた日常） |
| 縁側 | 現在（この瞬間だけが確かなもの） |
| 玄関 | 外の世界・別れと再会 |
| 寝室 | 一人になった後の時間 |
| 居間 | 家族の場（今は武一人） |

## 物語の骨格（最終）

**物語の前提**：

今朝、娘・恵から電話があった——たぶん。
恵が今日来ると言った——たぶん。
だから武は、家を「ちゃんとした状態」にしようとしている。

これがゲームの全体構造だ。
武がすることは「来るかもしれない娘のために、家を整える」こと。
これ以外の事件は起きない。

**登場人物**：
- 宮本 武（82歳）：プレイヤーが操作する視点人物
- 宮本 良子（故人）：フラッシュバック内に登場。武の妻
- 宮本 恵（推定50歳前後）：ドアベル音と声のみ登場。顔は見えない
- 教え子（男子・フラッシュバック内）：名前は明示されない

---

### 4つのフラッシュバック（最終版）

各フラッシュバックはキーオブジェクトに触れることで発動する。
フラッシュバック内でのみ選択肢が生まれる。
「どの選択が正解か」は設計上存在しない。

---

**フラッシュバック 1：書斎の油絵**（発動：書斎の壁に掛かった油絵）

武が30代だった頃。美術の授業後。
教え子の男子が武の絵を見ている。

```
武：「どう思う」

教え子：「……この絵は、何かを隠していますよね」

（武はその言葉を聞いて止まる）

プレイヤーの選択：
  A：「そうかもしれない」と答える
  B：「隠していない」と答える
  C：何も答えず、絵を直す

後日、別の教え子が言った——「先生の絵は嘘をつかない」
どちらが本当のことを言ったのか、武は今でも答えを持っていない。
```

「残るもの」：
- A：「認めた」という記憶。何を認めたのかは分からない
- B：「否定した」という記憶。何を守ろうとしたのかは分からない
- C：「直した絵」という記憶。言葉ではなく、手が動いた

---

**フラッシュバック 2：台所の赤いヤカン**（発動：ラベルのないヤカンを持ち上げる）

良子が死ぬ3ヶ月前の朝。
良子がヤカンでお湯を沸かしながら鼻歌を歌っている。
武はその歌の名前を知らない。

```
良子の鼻歌が終わる。武は——

プレイヤーの選択：
  A：「その歌、なんていう名前だい」と聞く
  B：一緒に歌おうとして、音程が外れる
  C：何も言わず、お茶を受け取る

良子はどの選択でもにっこりして、その朝は続く。
```

「残るもの」：
- A：歌の名前は今も分からない。でも「聞こうとした」が残る
- B：音程の外れた笑い声が残る
- C：温かいお茶の重さだけが残る

*注：このフラッシュバックから戻ると、台所のラベルが一段階変化している*

---

**フラッシュバック 3：縁側の植木鉢**（発動：縁側の色褪せた植木鉢）

恵が6歳だった頃。縁側で遊んでいた恵が転んで膝を擦りむいた。
武は絵の具の赤を取り出した。

```
泣いている恵の膝に、武は絵の具で小さな絵を描いてやった。

武：「きずは赤い。だから赤で描いてやった。
     でも本当は、きずは痛い色をしている。
     赤はぜんぶの痛みの色だから、ちょうどいい」

恵が泣き止んで、膝の絵を見ている。

プレイヤーの選択：
  A：「痛かったな」と言う
  B：「もう一個描こうか」と言う
  C：ただ、恵の頭を撫でる
```

「残るもの」：
- A：恵の「うん」という声
- B：恵の膝に増えた小さな絵たち
- C：頭の重さと温かさ

---

**フラッシュバック 4：玄関の鍵**（発動：玄関の鍵——ゲーム後半のみ発動）

恵が結婚して家を出た日。
武は「行ってらっしゃい」を言いそびれた。

```
恵がドアを閉めた後、武は玄関に立っていた。
何と言ったのかを覚えていない。
玄関の鍵だけが、その日と同じ場所にある。

プレイヤーの選択（武の「今の声」として）：
  A：「行ってらっしゃい」（言いそびれた言葉を、今ここで言う）
  B：「帰っておいで」（その日に言えなかったことを言う）
  C：（何も言わず、鍵を持つ）
```

このフラッシュバックだけは「過去の場面」が終わった後に、
武が現在の玄関に立ち、鍵を持ったまま止まるシーンが続く。

---

### 空のカンバス（特別インタラクション）

書斎に「空のカンバス」がある。
絵の具も、筆も、揃っている。

プレイヤーが「カンバスに触れる」を選択すると、武は筆を持つ。
しかし何も描けない。

武（内声）：「何を描くか、分かっていた。
　　　　　　 ずっと、分かっていた。
　　　　　　 今は、名前が出てこない。
　　　　　　 でも——形は分かる。色は分かる。」

筆が紙の上に触れる。
一本の線が引かれる。
武はそこで止まる。

プレイヤーには「描き続ける」「やめる」の選択肢が出る。

- 「描き続ける」：線だけの、形とも名前とも言えない何かが生まれる。これは結末には影響しない。ただそこにある
- 「やめる」：筆を置く。カンバスに一本の線だけが残る

---

### 結末（最終版）

ドアベルが鳴る。

プレイヤーは玄関へ向かう。
フラッシュバック4がまだ未発動なら、このとき発動する（強制発動）。

フラッシュバック4が終わると、武は現在の玄関に立っている。
鍵がある。

```
プレイヤーの選択：
  A：ドアを開ける
  B：「どなたですか」と声をかけてから開ける
  C：（もう少し待つ）——30秒後に自動でAに移行
```

どの選択でもドアは開く。
「お父さん」という声がする。

最後のシーン：縁側。武と恵がお茶を飲んでいる。

恵の顔は見えない。後ろ姿のみ。
武が縁側の向こうを見て、言う。

```
武：「きれいだな」

恵：「何が？」

武：「……ぜんぶ」
```

フェードアウト。

エンドロールなし。画面が暗くなったまま、縁側の環境音（風、遠くの鳥）が5秒続いてから終わる。

## ゲームプレイ設計（最終）

**UI方針**：

| 要素 | 設計 |
|---|---|
| ラベルの表示 | 手書きフォント。物に近づくとフワッと浮かび上がる。読まなくても進める |
| ラベルの変化 | 別の部屋を訪れて戻ると変化している。変化前との差をプレイヤーに強調しない |
| 操作制限の変化 | 序盤：引き出しはすべて開く。中盤：一部が「錆びついて」開かない。終盤：前に開けた引き出しが閉まっている |
| フラッシュバック | 画面が白く滲んでから切り替わる。BGMフェードアウト |

**「開かない引き出し」の前提開示**（世界観構築人・企画人の批評を受けた改訂）：

ゲーム冒頭、武の日記の一行が居間のテーブルに置かれている。
プレイヤーがそれを読む（任意）と：
```
「三月十四日（たぶん）
 昨日開けた引き出しが、今朝は開かなかった。
 中に何があったか、思い出せない。
 だから今日も同じ引き出しを開けようとしたのに。
 おかしい。でもしょうがない。」
```
これにより「操作制限が設計の一部」であることが物語の中で開示される。

**音響設計**：

| シーン | 音響 |
|---|---|
| 現在（武が歩く） | 環境音のみ（BGMなし）——時計の音、遠くの車、風 |
| フラッシュバック | 各シーンの環境音のみ（BGMなし）——子供の声、お湯の音、授業の雑音 |
| 空のカンバスのシーン | 完全な無音（3秒）、その後に風の音 |
| 結末・縁側 | 環境音のみ |

**全編を通じてBGMはない。**
「音楽を鳴らさないこと」が、この作品の音響設計の核心である。
音楽は感情を誘導する。この作品はプレイヤーの感情を誘導しない。
音楽がなくても、プレイヤーは感じる——それをSeep Logosは「没入感」と呼ぶ。

**感情属性の設計**：

各フラッシュバック選択肢の「残るもの」は等価に設計する。
優劣・正解・不正解を示す演出を一切禁止する。
選択後のフィードバックは「何が残ったかの静かな提示」のみ。

| 項目 | 内容 |

## エージェント間の論点と決着

### 論点1：「開かない引き出し」はバグに見えるか
**争点（企画人 → 世界観構築人）**：UI制限がプレイヤーに「バグか設計か」の混乱を招く可能性。
**決着**：日記の一行でルールを物語内に開示。採用確定。

### 論点2：選択肢の優劣
**争点（企画人 → シナリオ書き）**：「Cが最善に見える」リスク。
**決着**：選択後フィードバックの重みを均等化。選択肢の順番の設計も均等を意識する。「正解なし」を演出で担保。

### 論点3：油絵フラッシュバックへの美術感覚の追加
**争点（世界観構築人 → シナリオ書き）**：「嘘をつかない」という言葉に美術的文脈が薄い。
**決着**：教え子の「隠している」という言葉を前段に加え、武が美術教師として「見ること」に向き合った文脈を整合させた。

### 論点4：空のカンバスの追加
**争点（シナリオ書き → 企画人）**：「現在の武」がコアループに参加できていない。
**決着**：書斎に空のカンバスを配置。「描けないこと」を体験するインタラクションとして実装。結末への影響はなし。

### 論点5：ラベルのない赤いヤカン
**争点（シナリオ書き → 世界観構築人）**：ラベルのない物が一つあると象徴性が高まる。
**決着**：赤いヤカンをラベル未貼付の唯一の物として設計。台所のフラッシュバックとの接続により、物語の核心に昇格。

## プロデューサーへの判断依頼事項

**1. タイトルの方向性**
『Yesterday's Room』は英語タイトル。グローバル配信を前提とするなら有効。
国内限定なら日本語タイトル案（「昨日の部屋」「きれいだな、ぜんぶ」等）の検討を求める。

**2. PALIMPSESTとの関係**
v6はv5（PALIMPSEST）とは独立した企画として設計した。
2作を「Seep Logos 短編ゲームシリーズ」として同一ブランドで展開するか、それとも片方を優先するかの判断を求める。

**3. 音声の方針**
武の内声（ナレーション）を実際の声優音声にするか、テキストのみにするかで開発コストと没入感の設計が変わる。
「テキストのみ」案：コスト最小。プレイヤーが「武の声」を自分の中で作る。
「ナレーション音声あり」案：没入感は上がるが、キャスト費・収録費が発生する。

**4. 開発エンジンの確定**
Unity（URP）または Godot 4 を推奨。既存チームのスキルセットとの整合を優先する。

**5. Vertical Slice スコープの承認**
「台所・フラッシュバック2（赤いヤカン）」1シーン実装を最初のVertical Sliceとして推奨する。
承認の可否を求める。

## 全体戦略家所見

判断する。

v6は「記憶と忘却」というテーマを、テクノロジーも特殊職業も使わずに体験させる設計になった。
82歳の老人の一日という、最もシンプルな器の中に、最大の没入感を閉じ込める試みだ。

**v5との差別化**：
v5は「他者の記憶を鑑定する」知的・職業的体験だった。
v6は「自分の記憶が消えていく」身体的・感情的体験だ。
どちらもSeep Logosの「没入感」原則に合致しているが、アプローチは対照的だ。
2作を並走させることで、Seep Logosの短編ゲームとしての振り幅を示せる。

**最大の強度ポイント**：
「書斎に戻ったとき、ラベルが変化している」——この瞬間だ。
プレイヤーは説明なしに、身体で忘却を体験する。
言葉にする前に、体験が先に来る。これがSeep Logosの「没入感」だ。

**開発規模の評価**：
v5よりも開発コストが低く、実績として提示できる完成度に至る期間も短い。
「seeds」として探索を始め、Vertical Slice承認後に `titles/` へ昇格させる判断を推奨する。

**最後のセリフについて**：
「きれいだな」「ぜんぶ」——この2行が成立する理由は、武が美術教師だからだ。
形を見る人間が、最後まで形を見ている。
名前を失っても、美しさは残る。
このゲームはそのことを、一言の説明もなく伝える。
