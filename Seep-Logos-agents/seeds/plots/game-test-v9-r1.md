## 企画人（planning-developer） — Round 1 初稿

**ゲームタイトル案：「SANCTION / サンクション」**

**コンセプト一文：**
あなたは誰も殺していない。ただ、証明書を発行しただけだ。

**ジャンル：**
一人称ドキュメント推理 × 倫理的選択ゲーム（ノベル寄りのインタラクティブ体験）

**ターゲット：**
- 主軸：20〜35歳、社会問題に関心がある知識層ゲームプレイヤー
- 副軸：「Papers, Please」「This War of Mine」などの「苦い体験ゲーム」愛好者
- プラットフォーム：PC（Steam）、スマートフォン（縦持ち）

**ゲームの核心的問い：**
「正当な手続きを踏んだ行為は、倫理的に免責されるか？」

**プレイヤーの役割：**
倫理審査官（ECO）。自律型致死兵器（LAWS）に「倫理適合証明書」を発行する官僚。プレイヤーは申請書類を審査し、承認・却下・保留を選ぶ。

**ゲームサイクルの骨格：**
1. 申請書類が届く（LAWS型番・使用目的・予測死者数・担保措置）
2. 付随資料を読む（現地報告・遺族証言・兵器メーカー主張）
3. 判断を下す（承認 / 却下 / 条件付き承認）
4. 後日談が届く（判断の結果、何が起きたか）
5. 評価指標が更新される（審査精度・処理速度・省への貢献度）

**短編の射程：**
3〜5件の審査案件で完結。プレイ時間：90分以内。エンディング：3〜4種。

**参照作品：**
「Papers, Please」（手続きの倫理的暴力）、「Orwell」（監視と判断の責任）、「Disco Elysium」（政治的世界観のゲーム化）

## 世界観構築人（worldbuilding-creator） — Round 1 初稿

**世界設定：「サンクション」世界の構造**

**時代と場所：**
2041年。本作の舞台はハーグ（オランダ）。国際倫理審査機構（IEO: International Ethics Organization）の地方事務所。外では夕方の静かな街並み。内ではディスプレイとスキャナーの音だけが鳴る。

**「責任の空白」がどう制度化されているか：**

責任は以下の連鎖で分散される：

```
システム開発企業（設計上の問題ではない）
　→ 投資家（資金提供は適法）
　→ 軍（命令に従った）
　→ ECO（証明書を発行した）
　→ 指揮官（現場判断の範囲内）
　→ 政治家（法的手続きは完了）
　→ 市民（知らなかった）
```

どのリンクも「自分は正当な役割を果たした」と言える。そして死者は出る。

**「倫理適合証明書（ECC: Ethics Compliance Certificate）」の構造：**

- 発行者：ECO（倫理審査官）
- 審査対象：LAWS一台ごと（または型番ごと）
- 審査項目：MHC要件の充足、付随的損害予測値（CDV）の許容範囲内か、ロールバック機能の有無
- 有効期限：1年（更新制）
- 効果：証明書があれば、そのLAWSによる死者は「倫理審査済みの行為による結果」として行政処理される

**「意味ある人間の関与（MHC）」の形骸化：**

条文上の定義：「最終的な致死判断において人間の承認を経ること」
実態：「AIが選定したターゲットリストを人間が3秒以内に承認すること（タイムアウトはデフォルト承認）」

形式的には人間が決めている。実質的には人間は何も決めていない。

**「付随的損害（Collateral Damage）」の言語的機能：**

「死者」という言葉は公文書に登場しない。代わりに使われる用語：
- 付随的損害（CDV）：数値で表現される
- 非戦闘員への影響（NCI）：統計的に許容される誤差として処理
- 目標外接触事例（OTE）：後処理の対象

言葉が死を消している。

**主要用語集：**

| 用語 | 正式名称 | 実態 |
|---|---|---|
| LAWS | Lethal Autonomous Weapon System | 自律的に標的を選定・攻撃する兵器 |
| ECO | Ethics Compliance Officer | 証明書を発行する官僚 |
| MHC | Meaningful Human Control | 形骸化した「人間の関与」概念 |
| ECC | Ethics Compliance Certificate | 責任の空白を制度化する文書 |
| CDV | Collateral Damage Value | 死者数を行政的に無化する数値 |
| OTE | Off-Target Engagement | 「誤爆」の公式言い換え |
| IEO | International Ethics Organization | 国際倫理審査機構 |

## シナリオ書き（scenario-writer） — Round 1 初稿

**物語構造：「サンクション」**

**主人公：**
カイ・フェルメール（Kai Vermeer）、38歳。IEO（国際倫理審査機構）ハーグ事務所のECO（倫理審査官）。12年のキャリア。「仕事は正確に、感情は持ち込まない」が信条。

**物語の構造：3幕構成（短編）**

**第1幕：「仕事」（案件1〜2）**

案件1：東アフリカの農村地帯に展開予定の偵察型LAWS「IGNIS-3」の証明書更新申請。
- 書類は完璧に整っている。CDV予測値：0.3（基準値：2.0以下で承認）。
- プレイヤーは承認する（あるいはしない）。
- 後日談：IGNISは展開された。報告書には何も書かれていない。

案件2：より高い殺傷能力を持つ「KRONOS-7」の新規申請。
- CDV予測値：4.1（基準値超過）。しかし軍から「処理速度」の圧力がかかっている。
- プレイヤーが却下すると、上司から「再検討」を求められる。
- 承認すると、後日談で民間施設への「OTE」が報告される。

**第2幕：「証拠」（案件3）**

案件3（転換点）：突然、匿名のファイルが届く。
- 内容：過去にカイが発行した証明書リスト。それぞれの証明書の「後日談」（死者数、場所、日付）。
- カイが自分で承認した案件で、東アフリカの農村で87人が死んでいる。
- 「付随的損害」として処理されたその87人の名前と写真。
- カイは「殺した」のか。

**第3幕：「選択」（案件4——最後の審査）**

案件4：「NEXUS-プロトタイプ」の緊急申請。最高レベルの自律性を持つLAWS。
- 承認すれば、戦況を「終結」させる可能性がある（報告書上は）。
- 却下すれば、代わりに人間の兵士が戦場に投入される。
- カイはどちらも「殺すこと」に加担するのか。

**エンディング分岐：**

1. **「制度の内側」**：承認し続ける。カイはキャリアを全うする。戦争は続く。誰も問われない。
2. **「告発」**：全書類を内部告発機関に送る。カイは解雇され、訴追される。戦争は続く。
3. **「沈黙」**：最後の案件を承認も却下もしない。書類をデスクに置いたまま、カイは退職する。制度は続く。
4. **「問い直し」**（全案件を最低1回却下した場合のみ解放）：カイは証言台に立つことを選ぶ。法廷で、「誰が殺したか」を問う。答えは出ない。

**感情動線：**

```
平静（日常的な仕事）
　→ 違和感（数字の向こうに何かある）
　→ 直視（87人の名前と写真）
　→ 問い（自分は何者か）
　→ 選択（それでも生きる場所を決める）
```

**核心的セリフ（案件2の後、上司の言葉）：**
「フェルメール、君の仕事は法的整合性の確認だ。君は何も決めていない。君は確認しただけだ。」

## ゲームディレクター（game-director） — Round 1 初稿

**ゲームプレイ設計：「サンクション」**

**中核的な問い（ゲームデザイン視点）：**
「書類審査という退屈な行為が、なぜ恐ろしいのか」を体験させられるか。

**ゲームループ設計：**

```
【1サイクル = 1審査案件】
　↓
書類到着（テキスト＋スキャン画像風）
　↓
付随資料を読む（プレイヤー任意、読まなくてもよい）
　↓
判断入力（承認 / 却下 / 保留）
　↓
即時フィードバック：「処理完了」の無機質なスタンプ音
　↓
次の案件 or 後日談（遅延して届く）
　↓
【評価指標が更新される】
```

**3つのゲームメカニクス：**

**1. 「遅延後日談」システム：**
プレイヤーの判断の結果は、即座には届かない。案件1の後日談は案件3のあとに届く。この「ずれ」が責任の感覚を希薄にする。そして突然、87人の写真が届く。

**2. 「時間圧力」メカニクス：**
案件ごとに「処理期限」が設定されている。期限超過は評価指標を下げる。読めば読むほど時間が足りなくなる。「丁寧に読まないほうが仕事になる」という構造。プレイヤーは官僚の論理を内面化させられる。

**3. 「言語置換」システム：**
プレイヤーが「死者」と入力しようとすると、システムが「CDV」に自動変換する。ゲームのUIが「無害な言語」を強制する。プレイヤーはその都度、抵抗感を覚える。

**プレイヤーが「やめたくなる」設計と「続けたくなる」設計の共存：**

- やめたくなる要素：読めば読むほど加担している感覚。無機質なUI。後日談の重さ。
- 続けたくなる要素：「この案件は却下できるかもしれない」という僅かな希望。物語の続きを知りたい欲求。「自分だったらどうするか」の実験としての面白さ。

**難易度設計（短編ゲームとして）：**

難易度選択は不要。「正解がない」ことが設計の前提。チュートリアル代わりに、案件1は意図的に「よく整った書類」で始める。プレイヤーに「承認」させてから、その意味を後で問い直す。

**エンジニアリング上の推奨：**

- テキストベース。複雑な実装不要。
- UIの「無機質さ」自体がアートディレクション。
- BGMは環境音のみ（空調、書類をめくる音、スタンプ音）。
- プレイ時間：90分以内に収める。短編の説得力を守る。
