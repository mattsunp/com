## 企画人 (planning-developer) — IPコンセプト初稿

**タイトル案：「LETHE」（レーテー）**

ギリシャ神話の忘却の川「レーテー」に由来。死者が転生前に飲む水。忘れることで次へ進む、という逆説的な解放を核心に据える。

**コンセプトワード：** 忘れることは、喪失ではなく選択である。

**ジャンル：** ナラティブアドベンチャー（テキスト＋探索）

**プラットフォーム：** PC（Steam）／iOS・Android

**ターゲット：** 20〜35歳。喪失体験を持つ大人。「忘れたいのに忘れられない」感覚に共鳴できる層。

**IPコア：**
- 人は「忘れること」で前へ進む。しかし忘れた何かが自分の一部だったとしたら？
- プレイヤーは「記憶整理師」として他者の記憶に入り込む。整理（=削除）すべき記憶を選ぶことで物語が分岐する
- プレイヤー自身も「何かを忘れていく」ゲームシステムと連動する

**ビジネスモデル：**
- 買い切り（Steam：1,500円 / モバイル：980円）
- 短編3〜4時間。IP確立後にシーズン制展開

## 世界観構築人 (worldbuilding-creator) — 世界観初稿

**舞台：記憶の街「ムネーメー」**

現代と少しずれた都市。この世界では「記憶外科」という職業が存在する。病気や事故で記憶に障害を負った人、あるいは自ら「消したい記憶」を抱えた人が訪れるクリニックが街のそこかしこにある。

**世界のルール（設定三原則）：**

1. **記憶は物質として観測できる** — 専用機器「ニューロスコープ」を使えば、記憶は「光の残像」として視覚化される。鮮明なものは白く輝き、古びたものは黄ばみ、痛みを帯びたものは赤黒く滲む。

2. **忘却には副作用がある** — 記憶を消すと、それに紐づく感情・スキル・人間関係が連鎖して薄れる可能性がある。「あの日の出来事」を消したら、その日学んだことも、その日出会った人への愛着も、微かに揺らぐ。

3. **記憶は他者と共有できる** — 一部の人間は「記憶の読み取り・移植」が可能。これを「刻印師（しるしし）」と呼ぶ。主人公はこの能力を持つ。

**街の断面：**
- 「消去ローン」— 辛い記憶を消すための費用を分割払いにできる金融サービスが存在する
- 「記憶銀行」— 大切な記憶を外部に預けるサービス。預けた記憶は本人から切り離される
- 「追憶喫茶」— 他人の記憶を「体験コンテンツ」として鑑賞できる店。倫理的グレーゾーン

**主人公の立場：**
記憶外科クリニックに勤める若手の刻印師。仕事として他者の記憶に入り込み、「消すべきか残すべきかを判断する」補助を行う。ただし彼女自身の記憶には大きな欠落がある——なぜかは本人も知らない。

## シナリオ書き (scenario-writer) — 物語骨格初稿

**三幕構成：**

**第一幕「刻印」**
主人公・ナギは記憶外科クリニック「クロニクル」に勤める刻印師。ある日、依頼人・オルオが持ち込む——「妻との10年間の記憶を全て消してほしい」。妻は3ヶ月前に事故死。生き続けるために、忘れる必要があると言う。ナギは手続きに入るが、記憶の中で妻の「最後のメッセージ」を発見する。それは、オルオ本人も知らないメッセージだった。

**第二幕「剥離」**
消すか残すか——ナギは判断を保留し、オルオの記憶の奥へ踏み込んでいく。記憶の中に、オルオと妻の関係が「幸福ではなかった」事実が浮かぶ。しかし二人は確かに愛していた。「幸福でない愛」は消すべきか。ナギは迷いながら、自分自身の欠落した記憶が何かとリンクしていることに気づく。

**第三幕「忘却の選択」**
オルオの記憶の核心に辿り着く——妻が「記憶を消したいと願っていた」という事実。妻はナギのクリニックに来ていた。ナギが処理した記憶の中に、妻がいた。ナギはその事実を「覚えていない」——消したのか、消されたのか。プレイヤーはナギとして、三つの選択を迫られる：（A）オルオの記憶を消す、（B）最後のメッセージだけ残す、（C）全記憶を返還し、オルオ自身に選ばせる。

**感情動線の設計：**
- 序盤：他人事の静けさ（職業的距離感）
- 中盤：記憶に感情移入するほど、プレイヤー自身がナギと同期する
- 終盤：「忘れていたのは誰か」という問いが、プレイヤー自身に跳ね返る

## ゲームディレクター (game-director) — ゲームループ・体験設計初稿

**コアゲームループ：**

```
[依頼受諾] → [記憶探索] → [証拠収集・感情マッピング] → [判断] → [結果・連鎖変化]
```

**メカニクス設計「記憶層（レイヤー）システム」：**

記憶はUIとして3層構造で表現する。
- **表層（白）：** 依頼人が「覚えている」と思っている記憶。言語化された事実。
- **中層（黄）：** 感情と紐づいた断片。音・匂い・温度の記憶。言葉にならない記憶。
- **深層（赤黒）：** 本人が「見たくない」と蓋をした記憶。アクセスには「刻印師の負荷」が発生する。

深層に踏み込むほどナギ自身の記憶も揺らぐ——これをUIの「ノイズ」「文字の欠落」「選択肢の消失」として表現する。

**没入感の設計：**
- セーブができない区間を設ける。「記憶の中にいる間は戻れない」体験をUIで担保する
- プレイ時間に連動する要素を1点入れる。「時間が経つと記憶が薄れる（UIの輝度が落ちる）」——実装コスト懸念フラグあり
- 選択肢の文言が、プレイを重ねるにつれて少し変化する（ナギの記憶が変容しているため）

**プレイヤー体験の核心：**
「選択の重さ」ではなく「選択の後悔の設計」。プレイヤーが「正解を選んだと思った選択」が、後のシーンで「そうではなかった可能性」を見せる。後悔を「驚き」として機能させる。

**ジャンル確定意見：**
ナラティブアドベンチャーで問題ない。ただし「探索パート」の移動自由度を低くし、「記憶空間内の動き」に絞ることを推奨する。移動で没入が切れる。
