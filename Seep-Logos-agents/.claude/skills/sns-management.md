# SNS運用マニュアル（SNS Management）

## このスキルの目的
Seep Logos の公式SNSアカウントの運用方針・投稿ルール・危機管理対応を定めたマニュアル。
ブランドの一貫性を保ちながら、ファンコミュニティとの深い関係を築くための指針。

---

## 1. SNS運用の基本方針

### ミッションとSNSの接続
「無限の"世界"を創生する」というミッションをSNSで体現するために：
- 作品・世界観の魅力を継続的に発信する
- クリエイターの声・制作の熱量を伝える
- ファンとの双方向コミュニケーションを大切にする

### 発信の3原則
1. **世界観ファースト**: 世界観・IP の魅力を伝えることを中心に
2. **クリエイターへのリスペクト**: 関わる全クリエイターへの敬意を表現
3. **ファーストパーティ情報**: 二次情報より一次情報（制作現場からの発信）を重視

---

## 2. プラットフォーム別運用方針

### Twitter(X)
- **目的**: 情報発信・ファントレンド形成・リアルタイムコミュニケーション
- **投稿頻度**: 1日1〜3回（公式発表時は増加可）
- **文体**: 親しみやすくも品格ある。「です・ます」調を基本に自然な熱量を加える
- **推奨コンテンツ**: 新情報発表・開発こぼれ話・キャラクター誕生日・イベント情報
- **ハッシュタグ**: 公式タグ（作品名）＋ 場面別タグ（2〜5個を上限）

### Instagram
- **目的**: ビジュアル訴求・世界観浸透・海外ユーザーへのアプローチ
- **投稿頻度**: 週3〜5回（ストーリーズは毎日可）
- **文体**: 世界観重視のキャプション。長文も可（感情・背景を伝える）
- **推奨コンテンツ**: キャラクターイラスト・背景美術・コンセプトアート・制作風景
- **ハッシュタグ**: 最大30個（日本語タグ＋英語タグの組み合わせ）

### YouTube
- **目的**: PV・メイキング・ゲームプレイ動画の発信
- **投稿頻度**: 月2〜4本（キャンペーン期は増加）
- **概要欄**: タイトル・チャプター・関連リンク・クレジットを記載
- **推奨コンテンツ**: ゲームPV・キャラクタートレーラー・開発日記・公式生放送

### TikTok（展開する場合）
- **目的**: 若年層への認知拡大・バイラル展開
- **文体**: フレンドリー・テンポよく
- **推奨コンテンツ**: ショートトレーラー・キャラクター紹介・BGM紹介

## 3. 投稿文作成ルール

### 必須要素
- 情報の核心を冒頭に（「続きを読む」でカットされても伝わる）
- CTA（行動促進）を自然な形で組み込む
- 公式ハッシュタグは必ず入れる

### 禁止事項
- 絵文字の過剰使用（Twitter では2〜3個まで、Instagram では適度に）
- 「RT・いいねで拡散を！」等の強要的表現
- 他社作品との直接比較・批判
- 確定していない情報の断言
- 炎上ネタ・政治的・宗教的発言

### 投稿前チェック
- [ ] ブランドトーンに合っているか
- [ ] 未発表情報が含まれていないか
- [ ] 著作権・引用ルールに問題がないか
- [ ] 誤字脱字・数字の確認が完了しているか
- [ ] 画像・動画の使用許諾が確認されているか

## 4. 投稿カレンダー設計

### 月次カレンダーのテンプレート
```
【月次SNS投稿カレンダー（例）】

第1週：キャラクター紹介（キービジュアル or 立ち絵）
第2週：開発こぼれ話（制作者からのメッセージ）
第3週：世界観紹介（設定資料の一部公開）
第4週：イベント・キャンペーン情報

+ 随時：重要発表・お知らせ（最優先で挿入）
```

### 記念日・イベント日のコンテンツ設計
- キャラクターの誕生日：おめでとう投稿（イラスト付き）
- 作品の記念日：記念ビジュアル or ファン感謝メッセージ
- 業界イベント（TGS・AnimeJapan等）：出展情報・レポート

---

## 5. 危機管理・炎上対応の基本原則

### 初期対応（発生後1〜3時間以内）
1. 状況を正確に把握する（誰が・何を・どこで・どれだけの規模で）
2. 原因が自社にある場合、削除・修正を検討する
3. 対応方針を chief-producer ・ pr-communications と共有する
4. 「見えていないふり」は禁止。迅速な一次対応を取る

### 対応文章の原則
- 事実確認が取れていない段階での断言は避ける
- 感情的な反論はしない
- 「調査中」「確認中」でも一次対応は速やかに
- 謝罪が必要な場合は誠実に、しかし過度な謝罪はしない

## 6. よくあるミス & NG例

### NG例1：情報の先出し
```
NG: 公式発表前に制作メンバーが個人SNSで新作情報を言及
OK: 全スタッフへの情報管理ルール周知と、発表解禁日の明示
```

### NG例2：コピペ的な定型文
```
NG: 毎回同じフォーマットの「〇〇が更新されました」投稿
OK: そのコンテンツならではの「伝えたいこと」を一言添える
```
