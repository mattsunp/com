# プロジェクト管理マニュアル（Project Management）

## このスキルの目的
Seep Logos における全プロジェクトの進行管理・スケジュール設計・会議体運営の標準を定めたマニュアル。
クリエイティブな混沌の中にも、プロジェクトを確実に前進させる仕組みを作る。

---

## 1. プロジェクト管理の基本原則

### クリエイターの創造時間を守る
管理のための管理は不要。報告・会議・ドキュメント作成は最小限にし、クリエイターが「作る時間」を最大化する。

### 透明性
プロジェクトの状況（進捗・遅延・リスク）は全関係者がいつでも確認できる状態にする。

### 早期発見・早期対処
問題は小さいうちに発見し、対処する。「なんとかなると思っていた」が最大の失敗の原因。

---

## 2. プロジェクト立ち上げフロー

### Step 1: キックオフ前の準備
- プロジェクトの目的・ゴール・スコープを一枚の紙で定義する（プロジェクト憲章）
- ステークホルダーの一覧（誰が意思決定者か）を明確にする
- 初期スケジュール（マイルストーンのみ）を作成する

### Step 2: キックオフミーティング
- 目的：プロジェクトの全員の理解をそろえる
- アジェンダ：目的・ゴール・役割分担・スケジュール・コミュニケーション方法
- 成果物：議事録＋アクションアイテムリスト

### Step 3: WBS（作業分解）作成
- ゴールから逆算して必要な作業を全て洗い出す
- 各タスクに担当者・開始日・完了予定日を設定する
- クリティカルパス（遅延すると全体に影響するタスク列）を特定する

---

## 3. スケジュール設計の標準

### マイルストーン設計
```
【標準マイルストーン（ゲーム開発の例）】
M1: コンセプト確定
M2: プロトタイプ完成
M3: アルファ版完成（全機能実装）
M4: ベータ版完成（バグ修正・調整フェーズ）
M5: ゴールド版（マスターアップ）
M6: リリース
M7: ローンチ後レビュー
```

### バッファの確保
- 重要マイルストーン前には必ず「バッファ期間」を設ける（目安：工数の15〜20%）
- バッファは「消費していい時間」ではなく「使わないで済むように管理する予備」

### タスクの粒度
- 担当者1人が1〜3日で完了できる粒度に分割する
- 「〇〇の開発」（大きすぎ）→「〇〇機能のUI実装」（適切）

## 4. 進行管理の仕組み

### 週次ステータスレポート
```
## 今週の進捗サマリ（信号機評価）
🟢 問題なし  🟡 注意が必要  🔴 対応が必要

## 完了タスク
## 進行中タスク（進捗率付き）
## 遅延・リスク事項（原因と対策）
## 来週の予定
## 意思決定が必要な事項
```

### リスク管理
```
| リスク | 発生確率（高/中/低） | 影響度（高/中/低） | 対策 | 担当 | 期日 |
```
- 発生確率：高×影響度：高 → 即座に対応策を実施
- 発生確率：低×影響度：高 → コンティンジェンシープランを作成

### エスカレーションルール
- タスクの遅延が3日以上見込まれる → project-manager に報告
- スケジュール全体への影響がある → chief-producer に報告
- 予算超過・追加費用が発生 → chief-producer に即報告

---

## 5. 会議体の標準

### 週次定例（全体進捗確認）
- **目的**: 進捗確認・課題共有・意思決定
- **所要時間**: 30〜45分
- **参加者**: PM・各部門リード
- **準備物**: ステータスレポート（事前共有）

### 課題解決ミーティング（アドホック）
- **目的**: 特定の問題を解決する
- **所要時間**: 15〜30分
- **原則**: 意思決定者が必ず参加する

### 振り返り（レトロスペクティブ）
- **タイミング**: マイルストーン完了後 or プロジェクト終了後
- **内容**: よかった点・改善点・次に活かすこと
- **禁止**: 個人攻撃・犯人探し

## 6. よくあるミス & NG例

### NG例1：スコープクリープ
```
NG: 「ついでにこの機能も追加しよう」を繰り返し、スケジュールが崩壊する
OK: 追加要件は変更管理プロセスで評価し、スケジュール・予算への影響を確認してから承認する
```

### NG例2：バッファなしの計画
```
NG: 全タスクをパンパンに詰めた「余裕ゼロスケジュール」を作る
OK: マイルストーン前に必ず予備期間を設ける
```

### NG例3：報告が遅すぎる問題発覚
```
NG: 「なんとかなると思っていた」が続き、直前になって重大な遅延を報告する
OK: 「2日以上の遅延が見込まれたら即報告」のルールを全員で守る
```
