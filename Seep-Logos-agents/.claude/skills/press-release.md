# プレスリリース作成マニュアル（Press Release）

## このスキルの目的
Seep Logos から発信するプレスリリース・ニュースリリースの作成標準を定めたマニュアル。
記者が「使いたい」と思えるプレスリリースを毎回安定して作成するための手引き。

---

## 1. プレスリリースの基本原則

### 逆ピラミッド構造
```
最重要情報（5W1H）
    ↓
詳細・背景
    ↓
補足情報
    ↓
会社概要・問い合わせ先
```
最初の1〜2段落で要点を全て伝える。記者はリード文しか読まない場合がある。

### 記者の視点で書く
「この情報は読者に価値があるか？」「なぜ今なのか？」を常に問う。
発信者視点の「お知らせ」ではなく、読者視点の「ニュース」として設計する。

---

## 2. プレスリリース制作フロー

### Step 1: 情報収集
- 発表する事実（タイトル・日時・URL等）を全て収集
- 引用コメントが必要な場合、事前に確認・承認を取る
- 使用可能な素材（画像・動画）の有無を確認

### Step 2: ニュースアングルの設定
- 「なぜ今これがニュースになるか」を一文で言えるまで考える
- ターゲットメディアを3媒体以上想定し、刺さるアングルを設計

### Step 3: ドラフト執筆
- タイトル → リード文 → 本文 → 引用コメント → 概要 → 会社概要 の順で書く
- 全体700〜1,200字を目安

### Step 4: 社内確認
- 機密情報のチェック（未発表情報が含まれていないか）
- 数字・固有名詞の正確性確認
- 法務・コンプライアンス上の問題がないかの確認

### Step 5: 最終確認・配信
- 解禁日時の設定（エンバーゴ）
- 配信先（PR TIMES / PRTIMES / 個別メール等）の選定

---

## 3. 各パーツの書き方

### タイトル
- **目標**: 読んで内容が分かる。30字程度
- **構造**: [会社名/作品名] が [何をした/する] ── 具体的に
- **NG**: 「〇〇に関するお知らせ」（内容が分からない）
- **OK**: 「Seep Logos、新作RPG『〇〇』の正式タイトルと発売日を発表」

### リード文（最重要）
- **目標**: 5W1H を2〜3文以内に凝縮
- **構造**: [誰が] [何を] [いつ] [どこで] [なぜ] [どのように]
- **文字数**: 100〜150字程度
- **NG**: 会社の自己紹介から始める
- **OK**: 発表内容の核心から始める

### 本文
- 重要度の高い順に情報を並べる
- 1段落＝1テーマ（250字程度）
- 専門用語には必要に応じて注釈を付ける

### 引用コメント
- 氏名・役職を明記する
- 「リリースへの感想」ではなく「発表の背景・想い」を語る言葉にする
- 60〜120字程度

## 5. 品質チェックリスト

### 内容チェック
- [ ] 5W1H がリード文内で答えられているか
- [ ] 未発表情報・機密情報が含まれていないか
- [ ] 誇大広告表現（「業界初」等）に根拠があるか
- [ ] 引用コメントの氏名・役職が正確か

### 表記チェック
- [ ] タイトル・固有名詞の表記が正式名称か
- [ ] 数字（年月日・金額・版数等）に誤りがないか
- [ ] 誤字脱字がないか
- [ ] です・ます調で統一されているか

### 配信チェック
- [ ] エンバーゴ（解禁日時）が設定・共有されているか
- [ ] 問い合わせ先が正確か
- [ ] 添付素材（画像・動画）の使用権利は確保されているか

## 6. よくあるミス & NG例

### NG例1：自社目線のタイトル
```
NG: 「〇〇開発にあたり、チームが大切にした3つのこと」
OK: 「Seep Logos、新作ゲーム『〇〇』を本日リリース──世界観没入型RPGの新境地へ」
```

### NG例2：リード文に詳細情報を詰め込みすぎ
```
NG: 200字を超えるリード文に全情報を詰め込む
OK: 核心を2〜3文で、詳細は本文で展開する
```
