# シナリオ執筆マニュアル（Scenario Writing）

## このスキルの目的
Seep Logos のゲーム・アニメ・コミック等あらゆる媒体向けシナリオ執筆の標準手順・品質基準・テンプレートを定めたマニュアル。
このスキルを参照することで、誰が執筆しても「Seep Logos 品質」の物語が生まれる。

---

## 1. シナリオ制作の基本原則

### 世界観ファースト
シナリオは世界観設定（`worldbuilding.md` 参照）に基づいて執筆すること。
世界観と矛盾する展開・セリフ・設定は、必ず worldbuilding-creator に確認を取る。

### キャラクタードリブン
事件・ストーリーは「キャラクターの内的動機」から生まれること。
「都合の良い偶然」に頼らず、キャラクターの選択が物語を動かす。

### 感情のロードマップ
各シーンには「このシーンで読者/プレイヤーに感じさせたい感情」を必ず設定する。

---

## 2. シナリオ制作フロー

### Step 1: 企画・コンセプト確認
- 企画書（planning-developer 制作）と世界観資料（worldbuilding-creator 制作）を受け取る
- ジャンル・ターゲット・媒体（ゲーム/アニメ/コミック）を確認
- 制作物の種類（メインシナリオ / サブシナリオ / キャラクターエピソード等）を確認

### Step 2: ハコ書き（粗筋・構成案）
- 全体の三幕構成（起→承→転→結）を箇条書きで作成
- 各章・各話のサブタイトル・目的・感情目標を設定
- 主要な分岐ポイント（ゲームの場合）を設計

### Step 3: ト書き・セリフ執筆
- シーンのト書きを先に書いてから、セリフを乗せる
- キャラクター固有の言語パターン（口調・語彙・口癖）を一覧表で管理

### Step 4: 推敲・品質チェック
- 本チェックリスト（後述）で自己点検
- worldbuilding-creator に世界観整合確認を依頼

### Step 5: 最終稿
- バージョン番号と更新日を記録
- 承認者のサインオフを受けて確定稿とする

---

## 3. 三幕構成チェックリスト

### 第一幕（起）
- [ ] 主人公と世界の状況が伝わるか
- [ ] 読者/プレイヤーが感情移入できるキャラクター設定があるか
- [ ] 「普通の日常」から「物語の始まり」への転換点が明確か

### 第二幕（承・転）
- [ ] 主人公の目標と障壁が明確か
- [ ] 仲間・敵・事件が有機的に絡み合っているか
- [ ] 最も高い感情的緊張（クライマックス）が設計されているか

### 第三幕（結）
- [ ] 主要な伏線が回収されているか
- [ ] キャラクターが成長/変化しているか
- [ ] 読者/プレイヤーが満足感を得られるエンディングか

## 4. キャラクター設計テンプレート

```
【キャラクター名】
・読み方：
・年齢・性別・種族：
・役割（主人公 / ヒロイン / 敵役 / 脇役）：
・外見の特徴（ビジュアル担当が参照）：
・性格：
・口調・語尾・口癖：
・動機（何を求めているか）：
・秘密（本人が言えないこと）：
・成長弧（物語を通じてどう変化するか）：
・他キャラクターとの関係性：
```

## 5. ゲームシナリオ特有のルール

### 分岐シナリオの管理
- 分岐はフラグ名で管理（例：`FLAG_GOOD_ENDING`）
- 分岐ごとの結末と矛盾がないかチェック
- プレイヤーの選択が「意味を持つ」分岐設計を心がける

### セリフの文字数
- 1吹き出しは原則40〜60字以内（UI枠に収まる量）
- ナレーションは80字程度まで
- 長いセリフは複数の吹き出しに分割

### ボイス収録を意識したセリフ
- 声に出して読んで自然なリズムであること
- 息継ぎの位置を意識した句読点の配置
- 記号・特殊文字はボイス原稿に影響しないよう注意

## 6. よくあるミス & NG例

### NG例1：キャラクターの一貫性崩壊
```
NG: 「冷静沈着な設定のキャラが、突然感情的に泣き崩れる（理由なし）」
OK: 「感情を押し殺してきたキャラが、特定の出来事により初めて涙を見せる（段階的描写あり）」
```

### NG例2：説明的すぎるセリフ（説明台詞）
```
NG: 「そうだ、お前のことは知っている。お前は3年前に村を焼いた賞金稼ぎで、今は逃亡中だったな」
OK: 「……3年前の火の匂いが、今もお前から消えない」（示唆的な表現）
```

### NG例3：世界観無視の現代語
```
NG: 中世ファンタジー世界で「LINEしとく」「ヤバい」などの現代語
OK: 世界観に沿った言葉・表現に統一する
```

### NG例4：伏線の未回収
```
NG: 第1話で重要そうに語られた「封印の鍵」が結末で触れられない
OK: 必ず伏線管理表を作り、回収タイミングを管理する
```
